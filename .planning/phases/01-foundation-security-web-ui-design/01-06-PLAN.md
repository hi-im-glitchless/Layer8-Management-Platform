  <action>
    **User management routes (`backend/src/routes/users.ts`):**
    All routes require `requireAdmin` middleware + `auditMiddleware`.

    `GET /api/users` — List all users
    - Returns: `{ users: [{ id, username, isAdmin, isActive, totpEnabled, createdAt, updatedAt }] }`
    - Excludes passwordHash and totpSecret from response
    - Sorted by createdAt desc

    `POST /api/users` — Create user (auditMiddleware('admin.user.create'))
    - Accept: `{ username, password, isAdmin? }`
    - Validate: username 3-50 chars, alphanumeric + underscores, unique
    - Validate: password min 8 chars
    - Hash password with Argon2
    - Set `mustResetPassword: true`
    - Return created user (without hash)

    `PUT /api/users/:id` — Update user (auditMiddleware('admin.user.update'))
    - Accept: `{ username?, isAdmin?, isActive? }`
    - Cannot modify own admin status (prevent self-demotion)
    - Return updated user

    `POST /api/users/:id/reset-password` — Reset password (auditMiddleware('admin.user.password-reset'))
    - Accept: `{ password }`
    - Hash new password
    - Set `mustResetPassword: true` (user must change on next login)
    - Return `{ success: true }`

    `POST /api/users/:id/reset-totp` — Reset TOTP (auditMiddleware('admin.user.totp-reset'))
    - Set `totpSecret: null, totpEnabled: false`
    - Delete all trusted devices for this user
    - User will be forced to set up TOTP again on next login
    - Return `{ success: true }`

    `DELETE /api/users/:id` — Delete user (auditMiddleware('admin.user.delete'))
    - Cannot delete self
    - Hard delete (cascade deletes trusted devices, audit logs keep userId as null via SetNull)
    - Return `{ success: true }`

    **Admin routes (`backend/src/routes/admin.ts`):**
    All routes require `requireAdmin` middleware.

    `GET /api/admin/sessions` — List active sessions
    - Use session service's `getActiveSessions()` from Plan 03
    - Return `{ sessions: [{ sessionId, userId, username, ipAddress, lastActivity, createdAt }] }`

    `DELETE /api/admin/sessions/:sessionId` — Terminate session (auditMiddleware('admin.session.terminate'))
    - Use session service's `terminateSession()`
    - Cannot terminate own session
    - Return `{ success: true }`

    `POST /api/admin/sessions/cleanup` — Cleanup expired sessions (auditMiddleware('admin.session.cleanup'))
    - Delete expired sessions from Redis
    - Delete expired trusted devices from database
    - Return `{ sessionsCleared: N, devicesCleared: M }`

    **Template injection scanning (`backend/src/services/templateScan.ts`):**
    Defense-in-depth, low priority. Simple pattern matcher.

    `scanForInjection(content: string): { safe: boolean, findings: Finding[] }`:
    - Scan text content for dangerous Jinja2 patterns:
      - `{{ ... }}` expressions containing: `__class__`, `__mro__`, `__subclasses__`, `__globals__`, `__builtins__`, `__import__`, `config`, `self._`, `lipsum`, `cycler`, `joiner`, `namespace`
      - `{% ... %}` blocks containing: `import`, `include`, `extends` with user-controlled paths
      - Raw Python exec/eval patterns: `os.popen`, `subprocess`, `exec(`, `eval(`
    - Return list of findings with: pattern matched, line number, severity (high/medium/low)
    - This is NOT blocking — just informational scanning for uploaded templates

    Write basic tests in `backend/tests/services/templateScan.test.ts`:
    - Safe content passes: `{{ client_name }}`, `{% for finding in findings %}`
    - Dangerous patterns caught: `{{ ''.__class__.__mro__ }}`, `{% import 'os' %}`

    **Seed script (`backend/src/scripts/seed.ts`):**
    - Create a default admin user: username=`admin`, password=`Admin123!`, isAdmin=true, mustResetPassword=true
    - Add script to package.json: `"db:seed": "tsx src/scripts/seed.ts"`
    - Idempotent — skip if admin user already exists

    **Mount routes in `backend/src/index.ts`:**
