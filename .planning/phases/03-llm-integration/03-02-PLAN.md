    1. Install streamdown:

       cd frontend && npm install streamdown

    2. Create frontend/src/lib/llm-api.ts:
       - Uses the existing API_BASE_URL pattern from lib/api.ts (`import.meta.env.VITE_API_URL || 'http://localhost:3001'`)
       - `startLLMStream(prompt, options?)`: Initiates a POST to `/api/llm/generate` with prompt in body
         - This is NOT a simple fetch -- SSE over POST requires special handling
         - Use fetch() with POST method, include credentials, CSRF token, Content-Type: application/json
         - Read CSRF token using same getCsrfToken() pattern from lib/api.ts (extract from cookie)
         - Ensure CSRF token exists before request (same ensureCsrfToken pattern)
         - Returns a ReadableStream or the fetch Response object for the hook to consume
         - The response will be `text/event-stream` content type
         - Accept AbortSignal for cancellation
       - NOTE: Cannot use native EventSource for POST requests (EventSource only supports GET). Must use fetch + manual SSE parsing, or use a POST-capable SSE approach. The hook will parse the `text/event-stream` response manually.

    3. Create frontend/src/hooks/useStreamingLLM.ts:
       - Exports `StreamingState` interface: `{ content: string; isStreaming: boolean; isWaiting: boolean; error: string | null; usage?: { inputTokens: number; outputTokens: number } }`
         - `isWaiting`: true after stream starts but before first text chunk (for pulsing dots)
         - `isStreaming`: true while actively receiving chunks
       - `useStreamingLLM()` hook returns:
         - `state: StreamingState` (current streaming state)
         - `startStream(prompt: string, options?: { systemPrompt?: string; maxTokens?: number })`: Triggers generation
         - `stopStream()`: Cancels active stream via AbortController
         - `retry()`: Re-runs last prompt
       - Implementation:
         - Store AbortController in useRef (not useState -- prevent re-creation on re-render)
         - Store last prompt in useRef for retry
         - On startStream: reset state, set isWaiting=true, call startLLMStream()
         - Parse SSE response manually: read response body as stream, parse `event:` and `data:` lines
           - `event: delta` + `data: {"text":"..."}` -> append text to content, set isWaiting=false, isStreaming=true
           - `event: done` + `data: {"usage":{...}}` -> set isStreaming=false, store usage
           - `event: error` + `data: {"message":"...","retryable":true}` -> set error, isStreaming=false
         - On AbortController signal: close stream, set isStreaming=false (keep partial content)
         - Cleanup on unmount: abort any active stream
         - NOTE: Keep partial content visible on error (per user decision -- mid-stream failure keeps partial output)
  </action>
