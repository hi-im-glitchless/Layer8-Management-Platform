# Phase 01.1: UI/UX Visual Polish - Context

**Gathered:** 2026-02-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Visual refinement of the existing Phase 1 UI. Covers dark theme palette, logo integration, login page aesthetics, and component polish across both themes. No new functionality — purely how the app looks and feels.

</domain>

<decisions>
## Implementation Decisions

### Dark theme palette
- Deep dark backgrounds (near-black, like VS Code/Discord) — high contrast, dramatic
- Cool blue accent color for interactive elements (buttons, links, active states)
- Surface hierarchy uses a subtle mix: slight elevation (lighter grays) for raised elements + subtle borders where needed
- Both dark and light themes get equal attention and polish

### Logo & branding
- Two logo files provided: `layer8_logo_dark.png` (for dark backgrounds) and `layer8_logo_white.jpg` (for light backgrounds)
- Logos must blend seamlessly with backgrounds — look like integrated text, not a boxed image (no visible background box)
- Logo placement: sidebar top area + login page
- When sidebar is collapsed, logo hides completely
- Favicon derived from the Layer8 logo

### Login page aesthetics
- Atmospheric feel — not minimal, not terminal-inspired
- Split-screen layout retained: gradient side + form side
- Non-form side: dark gradient with subtle color shift (blue/purple tones matching the cool-blue accent)
- Form side: glassmorphism card — semi-transparent with backdrop blur over a shared background
- TOTP verification screen uses the same split + glassmorphism style for consistency

### Component refinement
- Sidebar: roomy & readable — more padding, clear labels, comfortable spacing (Notion-like)
- Sidebar nav items: background highlight on hover + left blue border accent on active item
- Tables (admin panel, audit logs): clean uniform rows with hover highlight — no striped rows
- Primary buttons: subtle blue gradient — stands out from the dark background
- Danger buttons: Claude's discretion on exact treatment

### Claude's Discretion
- Exact color hex values within the deep-dark + cool-blue direction
- Light theme color palette (should be polished and cohesive, matching the same accent direction)
- Danger button styling
- Transition/animation timing
- Exact glassmorphism opacity and blur values
- Gradient exact colors and direction on login page
- Typography refinements (sizes, weights, letter-spacing)
- Favicon cropping/derivation approach from the logo files

</decisions>

<specifics>
## Specific Ideas

- Logo files blend with background — white logo on light, dark logo on dark, no visible bounding box
- Login gradient should use blue/purple tones that tie into the cool-blue accent color system
- Glassmorphism on login form: semi-transparent card with backdrop-filter blur
- Sidebar active state: both background highlight AND left border accent (not one or the other)

</specifics>
