<objective>
Build the core sanitization pipeline: custom pentest recognizers (IP, hostname, AD objects, network paths, domains), deny list pre-processing, language detection, the custom mapping operator for consistent typed placeholders, and the sanitize/desanitize API endpoints.

Purpose: This is the heart of the sanitization system. It combines Presidio's built-in NER for standard PII (names, emails, phones, etc.) with custom regex/rule-based recognizers for pentest-specific entities. The deny list pre-processing ensures admin-specified terms are always caught. The mapping operator guarantees consistent, reversible, typed placeholders.

Output: Working POST /sanitize and POST /desanitize endpoints that accept text, detect all PII types, return sanitized text with typed placeholders, and can reverse the process.
</objective>

    **Custom Recognizers** (all extend `presidio_analyzer.PatternRecognizer`):

    **ip_address.py -- IPAddressRecognizer:**
    - Entity type: `IP_ADDR`
    - Pattern: IPv4 regex matching `\b(?:25[0-5]|2[0-4]\d|[01]?\d\d?)\.(?:25[0-5]|2[0-4]\d|[01]?\d\d?)\.(?:25[0-5]|2[0-4]\d|[01]?\d\d?)\.(?:25[0-5]|2[0-4]\d|[01]?\d\d?)\b` with base score 0.6
    - Context words: ["server", "host", "IP", "address", "network", "subnet", "target", "scan", "port"]
    - Override `validate_result` to REJECT:
      - Localhost range: 127.x.x.x
      - RFC5737 documentation: 192.0.2.x, 198.51.100.x, 203.0.113.x
      - Version strings: if preceded by common version indicators like "OpenSSH ", "Apache/", "nginx/", "v", "version " (look back ~20 chars before the match)
      - Link-local: 169.254.x.x
    - ACCEPT with boosted score (0.85) if in pentest context (surrounding words like "target", "host", "scan")
    - Also add IPv4 with CIDR pattern: `/\d{1,2}` suffix -- entity type still `IP_ADDR`, score 0.7

    **hostname.py -- HostnameRecognizer:**
    - Entity type: `HOSTNAME`
    - Patterns for internal FQDNs:
      - `\b[a-zA-Z0-9](?:[a-zA-Z0-9-]{0,61}[a-zA-Z0-9])?\.(?:local|internal|corp|lan|intranet|ad|domain)\b` (score 0.7)
      - `\b[a-zA-Z0-9](?:[a-zA-Z0-9-]{0,61}[a-zA-Z0-9])?(?:\.[a-zA-Z0-9](?:[a-zA-Z0-9-]{0,61}[a-zA-Z0-9])?)*\.(?:local|internal|corp|lan|intranet)\b` for multi-level FQDNs (score 0.75)
    - Context words: ["hostname", "server", "host", "DNS", "resolve", "domain", "FQDN"]
    - Validate: reject if term is a known product name or common word (maintain small exclusion list)

    **ad_objects.py -- ActiveDirectoryRecognizer:**
    - Entity type: `AD_OBJECT`
    - Pattern: `\b(?:CN|OU|DC)=(?:[^,\\]|\\.)+(?:,(?:CN|OU|DC)=(?:[^,\\]|\\.)+)*` (score 0.8)
    - Context words: ["Active Directory", "LDAP", "DN", "distinguished name", "domain controller", "OU", "organizational unit"]
    - High base score because AD DN format is very specific with low false positive rate

    **network_paths.py -- NetworkPathRecognizer:**
    - Entity type: `NETWORK_PATH`
    - Patterns:
      - UNC paths: `\\\\[a-zA-Z0-9._-]+\\[a-zA-Z0-9$._-]+(?:\\[a-zA-Z0-9._-]+)*` (score 0.8)
      - SMB URLs: `smb://[a-zA-Z0-9._-]+/[a-zA-Z0-9._/-]+` (score 0.8)
    - Context: ["share", "network", "UNC", "SMB", "mount", "file server"]

    **domain.py -- ExternalDomainRecognizer:**
    - Entity type: `DOMAIN`
    - Pattern: match common TLDs like .com, .net, .org, .io, .pt, .eu, .co.uk etc. with hostname prefix
    - Regex: `\b[a-zA-Z0-9](?:[a-zA-Z0-9-]{0,61}[a-zA-Z0-9])?\.(?:com|net|org|io|pt|eu|co\.uk|de|fr|es|it|nl|be|gov|edu|mil)\b` (score 0.5)
    - Context: ["domain", "website", "URL", "site", "client"]
    - Lower base score because domains have higher false positive rate -- context boosts to 0.7+
    - Validate: reject well-known non-sensitive domains (github.com, google.com, microsoft.com, etc.)

    **recognizers/__init__.py:**
    - Export a function `get_all_recognizers()` that returns a list of all custom recognizer instances

    **operators/mapping_replace.py -- MappingReplaceOperator:**
    - Maintain two dicts: `mappings: dict[str, dict[str, str]]` (entity_type -> {original_text: placeholder}) and `counters: dict[str, int]` (entity_type -> next_index)
    - `operate(text: str, params: dict) -> str`: Look up entity_type from params. If text already seen for this entity_type, return existing placeholder. Otherwise increment counter, create placeholder `[{ENTITY_TYPE}_{counter}]`, store mapping, return placeholder.
    - `get_forward_mappings() -> dict`: Returns {original_text: placeholder} flat dict for all entity types (for storage)
    - `get_reverse_mappings() -> dict`: Returns {placeholder: original_text} flat dict (for desanitization)
    - `load_mappings(forward: dict, counters: dict)`: Restore state from stored data
    - Make sure the placeholder format uses the entity_type directly: [PERSON_1], [IP_ADDR_2], [HOSTNAME_3], [CUSTOM_1], etc.

    **services/language_detector.py:**
    - `detect_language(text: str) -> tuple[str, float]`: Uses fast-langdetect on first 500 chars. Returns (language_code, confidence). Maps detected language to supported set: "en" or "pt". Defaults to "en" if unsupported language or low confidence (<0.7). Logs warning on low confidence.
    - `select_nlp_model(text: str, nlp_models: dict) -> tuple[Any, str]`: Calls detect_language, returns (spacy_model, language_code).

    **services/deny_list.py -- DenyListMatcher:**
    - `__init__(self, terms: list[str])`: Build compiled regex with word boundaries `\b` and case-insensitive flag from all terms. Escape special regex chars in terms.
    - `find_matches(self, text: str) -> list[tuple[str, int, int]]`: Returns (matched_text, start_offset, end_offset) for all matches
    - `to_recognizer_results(self, text: str) -> list`: Convert matches to Presidio `RecognizerResult` objects with entity_type="CUSTOM", score=1.0 (deny list matches are always high confidence)
    - Handle empty terms list gracefully (no matches, no regex compilation error)

    **services/sanitizer.py -- SanitizationService:**
    This is the core orchestrator. Initialize with nlp_models dict reference.

    `sanitize(text: str, deny_list_terms: list[str], language: str | None = None, entities: list[str] | None = None) -> SanitizeResult`:
    1. **Language detection**: If language not provided, auto-detect using language_detector. Select appropriate spaCy NLP model.
    2. **Deny list pre-processing**: Create DenyListMatcher with provided terms. Find all matches. Convert to RecognizerResult objects (entity_type="CUSTOM", score=1.0).
    3. **Presidio analysis**: Create AnalyzerEngine with the appropriate NLP engine (SpacyNlpEngine configured with the detected language model). Register all custom recognizers via `get_all_recognizers()`. Run `analyzer.analyze()` on the text.
    4. **Merge results**: Combine deny list RecognizerResults with Presidio results. For overlapping spans, deny list wins (higher score). Use Presidio's built-in result merging or implement span overlap resolution.
    5. **Apply anonymization**: Create MappingReplaceOperator instance. For each recognized entity, apply the operator to replace with typed placeholder. Handle the anonymization manually (iterate sorted results from end to start to preserve offsets) rather than using AnonymizerEngine with custom operator (Presidio's AnonymizerEngine custom operator API is complex -- manual replacement from end-to-start is simpler and more reliable for our typed placeholder format).
    6. **Build response**: Return sanitized text, list of DetectedEntity objects (with entity_type, start, end, score, original text, placeholder), detected language, entity counts by type, warning if no entities found.
    7. **Return mappings**: Return the forward mappings dict and counters dict from the operator (caller stores in Redis).

    `desanitize(text: str, reverse_mappings: dict[str, str]) -> DesanitizeResult`:
    1. Replace all placeholders in text with original values from reverse_mappings. Use regex `\[([A-Z_]+_\d+)\]` to find all placeholders.
    2. For each match, look up in reverse_mappings. If found, replace. If not found, add to unresolved list.
    3. After all replacements, scan for any remaining placeholder patterns -- if found, mark complete=False.
    4. Return desanitized text, complete flag, list of unresolved placeholders.

    Important implementation details:
    - The AnalyzerEngine should be created ONCE during service init (not per request) for performance. Register custom recognizers at init time. The NLP engine/language selection happens per-request via the `language` parameter of `analyzer.analyze()`.
    - Actually, Presidio AnalyzerEngine accepts `language` parameter in `analyze()` call, and it selects the appropriate NLP model. But we need to configure it with BOTH spaCy models. Use `NlpEngineProvider` with a config that maps "en" to en_core_web_lg and "pt" to pt_core_news_lg. See Presidio docs on multi-language support.
    - If the Presidio multi-language NLP engine setup is complex, an alternative is: create two AnalyzerEngine instances (one per language), select the right one based on detected language.
    - Score threshold: Filter results below the configured `default_confidence_threshold` (from config, default 0.5). But still include low-confidence results in the response with their scores -- flag them but don't exclude them from sanitization. The Phase 8 review UI will let users decide.

    **routes/sanitize.py:**
    - FastAPI APIRouter with prefix "" (mounted at root)
    - `POST /sanitize`: Accept SanitizeRequest body. Call sanitizer.sanitize(). Return SanitizeResponse with sanitized text, entities, language, entity counts, mappings data (forward_mappings, counters -- these are returned to the Node backend for Redis storage, NOT stored by the Python service).
    - `POST /desanitize`: Accept DesanitizeRequest body PLUS a `mappings` field (dict[str, str] -- reverse mappings, loaded from Redis by Node backend and forwarded). Call sanitizer.desanitize(). Return DesanitizeResponse.
    - Both endpoints return 503 if models not loaded (check app.state.models_loaded)
    - Both endpoints return 400 for validation errors (Pydantic handles this)
    - Add request logging: log entity counts and language for each request

    Update the request/response models if needed:

    - DesanitizeRequest should include `mappings: dict[str, str]` (reverse mappings from Redis)
