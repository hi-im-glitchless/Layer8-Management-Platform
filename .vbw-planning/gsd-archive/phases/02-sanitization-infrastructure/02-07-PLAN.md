The current desanitize() method in sanitizer.py (lines 201-261) uses `finditer` on the original text but `str.replace(..., 1)` on the mutating `desanitized_text`. This can produce incorrect results in edge cases:
- When an original value contains text that looks like another placeholder (e.g., original is "[SOME_TEXT_1]")
- When replace shifts positions and a subsequent replace targets the wrong occurrence

Replace the desanitize implementation with a single-pass `re.sub` approach that is both simpler and more correct:

```python
def desanitize(
    self,
    text: str,
    reverse_mappings: dict[str, str],
) -> DesanitizeResponse:
    import re

    unresolved_placeholders = []

    # Single-pass regex replacement using callback
    placeholder_pattern = re.compile(r'\[([A-Z_]+_\d+)\]')

    def replace_match(match):
        placeholder = match.group(0)
        if placeholder in reverse_mappings:
            return reverse_mappings[placeholder]
        else:
            if placeholder not in unresolved_placeholders:
                unresolved_placeholders.append(placeholder)
            return placeholder  # Leave unresolved as-is

    desanitized_text = placeholder_pattern.sub(replace_match, text)

    complete = len(unresolved_placeholders) == 0

    logger.info(
        f"Desanitization {'complete' if complete else 'incomplete'}, "
        f"unresolved: {len(unresolved_placeholders)}"
    )

    return DesanitizeResponse(
        text=desanitized_text,
        complete=complete,
        unresolved_placeholders=unresolved_placeholders,
    )
```

This approach:
1. Uses `re.sub` with a callback, which processes ALL matches in a single pass
2. No mutation of text between iterations -- each match is resolved independently
3. No position-tracking bugs -- regex engine handles offsets
4. Correctly handles multiple occurrences of same placeholder (each gets replaced)
5. Tracks unresolved placeholders in the callback
6. Removes the redundant "double-check" scan (the sub already handles everything)
7. Simpler code (fewer lines, fewer failure modes)

Keep the same method signature and return type. Keep the same logging format.
