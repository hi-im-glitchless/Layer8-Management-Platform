<objective>
Create the Node backend integration layer that bridges the Express API with the Python sanitization microservice. This includes an HTTP client for the Python service, Redis-based mapping storage with session-scoped TTLs, and proxy API routes that the frontend will call.

Purpose: The Node backend owns session management and Redis. The Python service is stateless -- it receives text + deny list terms, returns sanitized text + mappings. The Node backend stores mappings in Redis (keyed by session) and loads them for desanitization. This ensures mappings survive Python service restarts and are automatically cleaned up when sessions expire.

Output: Proxy routes at `/api/sanitize` and `/api/desanitize`, sanitization service with Redis mapping management, config update for Python service URL.
</objective>

  <name>Task 1: Create sanitization service with Python client and Redis mapping management</name>
  <files>
    backend/src/services/sanitization.ts

  </files>
  <action>
    **backend/src/config.ts** -- Add new env var:
    - `SANITIZER_URL`: string, default `http://localhost:8000` -- URL of the Python sanitization service

    **backend/src/services/sanitization.ts:**

    **Health check / readiness:**
    - `checkSanitizerHealth(): Promise<{ healthy: boolean, models_loaded: boolean, supported_languages: string[] }>` -- GET request to `${SANITIZER_URL}/health`. Parse JSON response. Return health status. Catches errors and returns `{ healthy: false, models_loaded: false, supported_languages: [] }` on connection failure.
    - `waitForSanitizer(maxWaitMs = 30000, intervalMs = 2000): Promise<boolean>` -- Poll health endpoint until models_loaded=true or timeout. Log progress. Used during server startup. Returns true if ready, false if timeout.

    **Sanitize:**
    - `sanitizeText(text: string, sessionId: string, denyListTerms: string[], options?: { language?: string, entities?: string[] }): Promise<SanitizeResult>`:
      1. POST to `${SANITIZER_URL}/sanitize` with `{ text, session_id: sessionId, deny_list_terms: denyListTerms, language: options?.language, entities: options?.entities }`
      2. Parse response to get sanitized_text, entities, language, entity_counts, mappings, counters
      3. Store mappings in Redis: key `layer8:mappings:${sessionId}`, value JSON-stringified `{ forward: mappings, reverse: reverseMap, counters }`. Compute reverse map by inverting the forward mappings.
      4. Set TTL on Redis key to match session TTL (read from config or use 30 days default matching session cookie maxAge)
      5. Return sanitized text, entities list, language, entity counts, and a warning string if applicable
    - Type `SanitizeResult`: `{ sanitizedText: string, entities: DetectedEntity[], language: string, entityCounts: Record<string, number>, warning?: string }`
    - Type `DetectedEntity`: `{ entityType: string, start: number, end: number, score: number, text: string, placeholder: string }`

    **Desanitize:**
    - `desanitizeText(text: string, sessionId: string): Promise<DesanitizeResult>`:
      1. Load mappings from Redis: key `layer8:mappings:${sessionId}`
      2. If no mappings found, throw error "No mappings found for session -- may have expired"
      3. Extract reverse mappings from stored data
      4. POST to `${SANITIZER_URL}/desanitize` with `{ text, session_id: sessionId, mappings: reverseMappings }`
      5. Parse response. Return desanitized text, completeness flag, unresolved placeholders.
    - Type `DesanitizeResult`: `{ text: string, complete: boolean, unresolvedPlaceholders: string[] }`

    **Mapping management:**
    - `getMappings(sessionId: string): Promise<{ forward: Record<string, string>, reverse: Record<string, string> } | null>` -- Load mappings from Redis for a session. Returns null if not found.
    - `deleteMappings(sessionId: string): Promise<boolean>` -- Delete mappings for a session. For cleanup.

    **Error handling:**
    - If Python service returns 503 (models not loaded), throw a specific error: "Sanitization service not ready -- models still loading"
    - If Python service is unreachable (ECONNREFUSED), throw: "Sanitization service unavailable"
    - If Python service returns 4xx/5xx, throw with the error message from the response body
    - Use native `fetch()` (available in Node 18+, which this project uses with Express 5)

    Import `redisClient` from `@/db/redis.js`. Import `config` from `@/config.js`.
  </action>

  <name>Task 2: Create proxy API routes and mount in Express app</name>
  <files>
    backend/src/routes/sanitization.ts

  <action>
    **backend/src/routes/sanitization.ts:**
    Create Express router with authenticated endpoints:

    - `POST /api/sanitize` -- Sanitize text.
      - Requires authentication (check req.session.userId)
      - Body (validate with zod): `{ text: string (required, max 500000 chars), language?: string, entities?: string[] }`
      - Load global deny list terms using the deny list service from Plan 02: `getAllActiveTerms()` from `@/services/denyList.js`. Also accept optional per-request deny list terms from the body and merge with global terms.
      - Actually, add `denyListTerms?: string[]` to the request body for per-session terms (Phase 8 UI will send these)
      - Merge: global active terms + request body terms (deduplicated)
      - Call `sanitizeText(text, req.session.id, mergedTerms, { language, entities })`
      - Return: `{ sanitizedText, entities, language, entityCounts, warning }`
      - Do NOT return the raw mappings to the frontend (security -- mappings contain original PII). The mappings are stored in Redis server-side.
      - Log audit: action "sanitize", details include entity counts and language (NOT the original text or entities)

    - `POST /api/desanitize` -- Desanitize text.
      - Requires authentication
      - Body: `{ text: string (required, max 500000 chars) }`
      - Call `desanitizeText(text, req.session.id)`
      - Return: `{ text, complete, unresolvedPlaceholders }`
      - Log audit: action "desanitize", details include complete flag and unresolved count

    - `GET /api/sanitize/health` -- Check Python service health.
      - Requires authentication
      - Call `checkSanitizerHealth()`
      - Return health status JSON

    - `GET /api/sanitize/mappings` -- Get mapping summary for current session.
      - Requires authentication
      - Load mappings from Redis for current session
      - Return entity count summary (NOT the actual mappings/original values): `{ hasMappings: boolean, entityCounts: Record<string, number> }`
      - This lets the frontend know if a session has active mappings without exposing PII

    **backend/src/index.ts:**
    - Import `sanitizationRouter` from `./routes/sanitization.js`
    - Mount: `app.use('/api/sanitize', sanitizationRouter);`
    - Note: The desanitize route is `POST /api/sanitize/desanitize` (nested under sanitize prefix). Alternatively, mount desanitize separately. Cleaner to use: `app.use('/api', sanitizationRouter);` and define routes as `/sanitize` and `/desanitize` in the router. Choose whichever is cleaner -- the key is that both endpoints are accessible.

    Optionally, during server startup in `startServer()`, call `waitForSanitizer()` with a reasonable timeout. If it fails, log a warning but DON'T prevent server start (the sanitization service might start later). Routes will return 503 if the service isn't ready when called.
