       ```bash
       npm install --save-dev @types/express @types/cors @types/express-session
       ```

    3. Configure `tsconfig.json`:
       - `target`: "ES2022"
       - `module`: "NodeNext"
       - `moduleResolution`: "NodeNext"
       - `outDir`: "./dist"
       - `rootDir`: "./src"
       - `strict`: true
       - `esModuleInterop`: true
       - `resolveJsonModule`: true

    4. Add scripts to `package.json`:
       - `"dev": "tsx watch src/index.ts"`
       - `"build": "tsc"`
       - `"start": "node dist/index.js"`
       - `"db:push": "prisma db push"`
       - `"db:migrate": "prisma migrate dev"`
       - `"db:studio": "prisma studio"`
       - Set `"type": "module"` for ESM

    5. Create `backend/src/config.ts` — Zod-validated env config:
       ```typescript
       import { z } from 'zod';

       const envSchema = z.object({
         NODE_ENV: z.enum(['development', 'production', 'test']).default('development'),
         PORT: z.string().default('3001').transform(Number),
         DATABASE_URL: z.string().default('file:./dev.db'),
         REDIS_URL: z.string().default('redis://localhost:6379'),
         SESSION_SECRET: z.string().min(32, 'SESSION_SECRET must be at least 32 characters'),
         FRONTEND_URL: z.string().default('http://localhost:5173'),
       });

       const parsed = envSchema.safeParse(process.env);
       if (!parsed.success) {
         console.error('Invalid environment variables:', parsed.error.format());
         process.exit(1);
       }
       export const config = parsed.data;
       ```

    6. Create `backend/src/db/redis.ts`:
       - Create Redis client using `createClient({ url: config.REDIS_URL })`
       - Error handler logging
       - Export `redisClient` and `connectRedis()` async function
       - Export `createRedisStore()` function returning `new RedisStore({ client: redisClient })`

    7. Create `backend/src/db/prisma.ts`:

       - Handle disconnect on process exit

    8. Create `backend/src/index.ts` — Express app entry point:
       - Load dotenv first (`import 'dotenv/config'`)
       - Import config (triggers Zod validation)
       - Set up Express with:
         - `express.json()` body parser

         - `express-session` with Redis store:

           - `resave`: false
           - `saveUninitialized`: false
           - `cookie`: httpOnly=true, secure=(NODE_ENV==='production'), maxAge=30 days, sameSite='lax'
       - Health check route: `GET /api/health` returns `{ status: 'ok', timestamp }`
       - Start server on `config.PORT`
       - Connect Redis before starting
       - Log startup with port number

  <action>
    Initialize Prisma and create the schema:

    ```bash
    cd backend && npx prisma init --datasource-provider sqlite
    ```

    Define models in `backend/prisma/schema.prisma`:

    **User model:**
    ```prisma
    model User {
      id            String    @id @default(cuid())
      username      String    @unique
      passwordHash  String
      isAdmin       Boolean   @default(false)
      isActive      Boolean   @default(true)
      mustResetPassword Boolean @default(true)
      totpSecret    String?
      totpEnabled   Boolean   @default(false)
      failedLoginAttempts Int  @default(0)
      lockedUntil   DateTime?
      createdAt     DateTime  @default(now())
      updatedAt     DateTime  @updatedAt

      trustedDevices TrustedDevice[]
      auditLogs      AuditLog[]
    }
    ```

    - `id`: cuid for non-sequential, URL-safe IDs
    - `mustResetPassword`: true when admin creates account (user must set their own password on first login)
    - `totpSecret`: nullable until TOTP setup is completed
    - `failedLoginAttempts` + `lockedUntil`: for account lockout policy (Claude's discretion: implement progressive lockout — lock for 5 min after 5 failures, 15 min after 10, require admin unlock after 15)
    - `isActive`: soft-disable accounts without deletion

    **TrustedDevice model:**
    ```prisma
    model TrustedDevice {
      id          String   @id @default(cuid())
      userId      String
      deviceHash  String
      expiresAt   DateTime
      createdAt   DateTime @default(now())

      user User @relation(fields: [userId], references: [id], onDelete: Cascade)

      @@index([userId])
      @@index([deviceHash])
    }
    ```

    - `deviceHash`: hash of user agent + IP or a random token stored in a cookie
    - `expiresAt`: 30 days from creation (per locked decision: TOTP "remember me" lasts 30 days)

    **AuditLog model:**
    ```prisma
    model AuditLog {
      id           String   @id @default(cuid())
      userId       String?
      action       String
      details      String   @default("{}")
      ipAddress    String
      previousHash String
      hash         String
      createdAt    DateTime @default(now())

      user User? @relation(fields: [userId], references: [id], onDelete: SetNull)

      @@index([userId])
      @@index([action])
      @@index([createdAt])
    }
    ```

    - `userId` is nullable: system events (startup, chain verification) have no user
    - `details`: JSON string (not JSON type — SQLite compatibility)
    - `previousHash` + `hash`: tamper-evident hash chain
    - Indexes on `userId`, `action`, `createdAt` for efficient querying and filtering
