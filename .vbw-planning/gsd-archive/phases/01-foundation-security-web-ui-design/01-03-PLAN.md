    **TypeScript session augmentation (`backend/src/types/express.d.ts`):**
    Augment `express-session` SessionData to include:
    - `userId: string`
    - `username: string`
    - `isAdmin: boolean`
    - `totpVerified: boolean` (true after TOTP verification OR trusted device)
    - `pendingTOTPSecret?: string` (during TOTP setup flow)
    - `awaitingTOTP?: boolean` (password verified, waiting for TOTP code)

    **Session service (`backend/src/services/session.ts`):**

    Write tests first (TDD) then implement:
    - `createTrustedDevice(userId: string, deviceIdentifier: string): Promise<string>` — creates TrustedDevice record with 30-day expiry, returns device token
    - `isTrustedDevice(userId: string, deviceToken: string): Promise<boolean>` — checks if valid (not expired) trusted device exists
    - `cleanupExpiredDevices(): Promise<number>` — deletes expired TrustedDevice records, returns count
    - `getActiveSessions(redisClient): Promise<SessionInfo[]>` — scans Redis for active sessions, returns user/IP/lastActivity
    - `terminateSession(sessionId: string, redisClient): Promise<void>` — destroys specific session in Redis

    **Rate limiting middleware (`backend/src/middleware/rateLimit.ts`):**
    - `loginRateLimiter`: 5 attempts per 5 minutes per IP using `express-rate-limit`
    - Use Redis store (`rate-limit-redis`) for persistence: `npm install rate-limit-redis`
    - `generalRateLimiter`: 100 requests per 15 minutes per IP
    - Return generic error message: "Too many requests. Please try again later."

    **CSRF middleware (`backend/src/middleware/csrf.ts`):**
    - Configure `csrf-csrf` with double-submit cookie pattern
    - Cookie name: `__csrf`
    - Skip CSRF for: GET, HEAD, OPTIONS requests
    - Apply to all state-changing routes (POST, PUT, DELETE, PATCH)
    - Frontend must read CSRF token from cookie and send in `X-CSRF-Token` header

    **Auth middleware (`backend/src/middleware/auth.ts`):**
    - `requireAuth`: checks `req.session.userId` and `req.session.totpVerified` are both truthy. Returns 401 if not.
    - `requireAdmin`: calls requireAuth first, then checks `req.session.isAdmin`. Returns 403 if not admin.
    - `requirePendingTOTP`: checks `req.session.awaitingTOTP` is true (user passed password but not TOTP yet). Returns 401 if not.

    **Auth routes (`backend/src/routes/auth.ts`):**

    All input validated with Zod schemas.

    `POST /api/auth/login` (rate limited):
    - Accept `{ username, password }`
    - Find user by username (return generic "Invalid credentials" if not found — prevents enumeration per locked decision)
    - Check account lock status. If locked, return generic "Invalid credentials" (do NOT reveal lock status)
    - Verify password with Argon2. If wrong, increment failed attempts, return "Invalid credentials"
    - Reset failed attempts on success
    - If user has `mustResetPassword: true`: set session flag and return `{ requiresPasswordChange: true }`
    - If user has `totpEnabled: true`:
      - Check if device is trusted (check cookie `layer8_trusted_device` against TrustedDevice table)
      - If trusted: set `totpVerified: true`, return `{ success: true }` with full session
      - If not trusted: set `awaitingTOTP: true`, return `{ requiresTOTP: true }`
    - If user has `totpEnabled: false` (first login, TOTP not yet set up):
      - Return `{ requiresTOTPSetup: true }`

    `POST /api/auth/login/totp` (rate limited, requirePendingTOTP):
    - Accept `{ code, rememberDevice?: boolean }`
    - Verify TOTP code against user's stored secret
    - Implement replay prevention: store used codes in Redis with 90-second TTL (key: `totp:used:{userId}:{code}`)
    - If valid:
      - Set `totpVerified: true`, clear `awaitingTOTP`
      - If `rememberDevice` is true: create trusted device, set `layer8_trusted_device` cookie (httpOnly, 30 days)
      - Return `{ success: true }`
    - If invalid: return 401 with "Invalid code"

    `POST /api/auth/totp/setup` (requireAuth but totpVerified not required — user is in setup flow):
    - Generate TOTP secret
    - Store secret in `req.session.pendingTOTPSecret`
    - Return `{ qrCodeDataURL }`

    `POST /api/auth/totp/verify-setup` (requireAuth):
    - Accept `{ code }`
    - Verify code against `req.session.pendingTOTPSecret`
    - If valid: save secret to user record, set `totpEnabled: true`, clear `pendingTOTPSecret`, set `totpVerified: true`
    - Return `{ success: true }`

    `POST /api/auth/password/change` (requireAuth or during first-login flow):
    - Accept `{ currentPassword (optional if mustResetPassword), newPassword }`
    - Validate new password: min 8 chars, at least 1 uppercase, 1 lowercase, 1 number
    - Hash new password, update user, set `mustResetPassword: false`
    - Return `{ success: true }`

    `POST /api/auth/logout`:
    - Destroy session (`req.session.destroy()`)
    - Clear session cookie
    - Return `{ success: true }`

    `GET /api/auth/me` (requireAuth):
    - Return current user info: `{ id, username, isAdmin, totpEnabled }`

    **Mount routes in `backend/src/index.ts`:**
    - `app.use('/api/auth', authRouter)`
    - Apply CSRF middleware globally (after session middleware)
    - Apply general rate limiter globally
