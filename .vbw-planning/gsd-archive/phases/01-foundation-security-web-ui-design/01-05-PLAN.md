    **RED — Write failing tests (`backend/tests/services/audit.test.ts`):**

    - Creates an audit log entry with all required fields (userId, action, details, ipAddress)
    - First entry has previousHash of genesis hash ("0".repeat(64))
    - Second entry's previousHash equals first entry's hash
    - Entry hash is a SHA-256 hex string (64 characters)

    Test cases for `verifyAuditChain`:
    - Returns `{ valid: true, entries: N }` for an untampered chain of 5+ entries
    - Returns `{ valid: false, brokenAt: id }` when a middle entry's hash is manually modified
    - Returns `{ valid: true, entries: 0 }` for an empty chain (no entries)

    Test cases for `queryAuditLogs`:
    - Returns all logs when no filters (paginated, default page size 50)
    - Filters by userId
    - Filters by action type (exact match)
    - Filters by date range (startDate, endDate)
    - Combines filters (userId + action + dateRange)
    - Returns results sorted by createdAt descending (newest first)
    - Pagination works (page, pageSize)
    - Returns total count for pagination UI

    Test cases for `exportAuditLogs`:
    - Returns JSON array of all matching logs
    - Each log includes: id, userId, username (joined), action, details (parsed JSON), ipAddress, createdAt, hash
    - Filters same as queryAuditLogs

    Test case for concurrent writes:
    - Submit 10 audit events simultaneously using Promise.all
    - Verify chain is still valid after all writes complete (no duplicate previousHash values)

    **GREEN — Implement `backend/src/services/audit.ts`:**

    ```typescript
    interface AuditEvent {
      userId?: string | null;
      action: string;
      details: Record<string, unknown>;
      ipAddress: string;
    }

    interface AuditQueryFilter {
      userId?: string;
      action?: string;
      startDate?: Date;
      endDate?: Date;
      page?: number;
      pageSize?: number;
    }
    ```

    `logAuditEvent(event: AuditEvent): Promise<void>`:
    - Use `prisma.$transaction()` with serializable isolation to prevent race conditions (per research pitfall #2)
    - Inside transaction:
      1. Find last entry ordered by createdAt desc (select only hash)
      2. Use genesis hash `"0".repeat(64)` if no previous entry
      3. Create entry string: `JSON.stringify({ userId, action, details, ipAddress, timestamp: new Date().toISOString() })`
      4. Compute hash: `crypto.createHash('sha256').update(previousHash + entryString).digest('hex')`
      5. Create AuditLog record with all fields

    `verifyAuditChain(): Promise<{ valid: boolean, entries: number, brokenAt?: string }>`:
    - Fetch all entries ordered by createdAt asc
    - For each entry starting from index 1:
      - Recompute hash from entry data + previous entry's hash
      - If computed hash !== stored hash, return `{ valid: false, brokenAt: entry.id }`
    - Return `{ valid: true, entries: count }`

    `queryAuditLogs(filter: AuditQueryFilter): Promise<{ logs: AuditLog[], total: number }>`:
    - Build Prisma where clause from filters
    - Use `prisma.auditLog.findMany()` with `skip`, `take`, `orderBy: { createdAt: 'desc' }`
    - Include user relation for username
    - Use `prisma.auditLog.count()` for total

    `exportAuditLogs(filter: Omit<AuditQueryFilter, 'page' | 'pageSize'>): Promise<ExportedAuditLog[]>`:
    - Same filtering as queryAuditLogs but no pagination
    - Include username from User relation
    - Parse details JSON string into object
    - Return flat array suitable for JSON export

    **REFACTOR:** Extract hash computation into a pure `computeEntryHash(entryString, previousHash)` function for testability.

    **Audit middleware (`backend/src/middleware/audit.ts`):**

    Create factory function that generates audit-logging middleware:
    ```typescript
    export function auditMiddleware(action: string) {
      return async (req: Request, res: Response, next: NextFunction) => { ... }
    }
    ```

    - Call `next()` first to let the request execute
    - On `res.finish` event (after response is sent), log the audit event asynchronously
    - Extract from request:
      - `userId`: from `req.session?.userId` (null for unauthenticated requests like login attempts)
      - `action`: from factory parameter
      - `details`: `{ method: req.method, path: req.path, statusCode: res.statusCode, body: sanitizeBody(req.body) }`
      - `ipAddress`: from `req.ip` or `req.headers['x-forwarded-for']`
    - `sanitizeBody(body)`: remove `password`, `currentPassword`, `newPassword`, `code`, `token` fields from body before logging (never log sensitive data)
    - Do NOT block the response on audit logging — fire and forget with error catch (log audit failure to console)

    **Apply audit middleware to auth routes (`backend/src/routes/auth.ts`):**
    Add `auditMiddleware()` to each auth route:
    - `POST /api/auth/login` → `auditMiddleware('auth.login')`
    - `POST /api/auth/login/totp` → `auditMiddleware('auth.totp.verify')`
    - `POST /api/auth/totp/setup` → `auditMiddleware('auth.totp.setup')`
    - `POST /api/auth/totp/verify-setup` → `auditMiddleware('auth.totp.complete')`
    - `POST /api/auth/password/change` → `auditMiddleware('auth.password.change')`
    - `POST /api/auth/logout` → `auditMiddleware('auth.logout')`

    **Audit API routes (`backend/src/routes/audit.ts`):**

    `GET /api/audit` (requireAuth):
    - Query params: `page`, `pageSize`, `action`, `startDate`, `endDate`
    - If user is admin: can see all logs, can filter by `userId`
    - If user is NOT admin: automatically filters to own logs only (`userId = req.session.userId`)
    - Returns `{ logs: [...], total: number, page: number, pageSize: number }`

    `GET /api/audit/export` (requireAdmin):
    - Same query params as above (minus pagination)
    - Returns JSON array with `Content-Disposition: attachment; filename="audit-log-{date}.json"`
    - Content-Type: `application/json`

    `GET /api/audit/verify` (requireAdmin):
    - Runs `verifyAuditChain()` and returns result
    - `{ valid: boolean, entries: number, brokenAt?: string }`

    **Mount in `backend/src/index.ts`:**

    - Log result: "Audit chain verified: {entries} entries, valid: {valid}"
    - If invalid, log warning but do NOT prevent startup
