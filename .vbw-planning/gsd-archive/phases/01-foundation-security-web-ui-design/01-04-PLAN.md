  <action>
    **API client (`frontend/src/lib/api.ts`):**
    - Create a typed fetch wrapper function:
      ```typescript
      export async function apiClient<T>(url: string, options?: RequestInit): Promise<T>
      ```
    - Always include `credentials: 'include'` for cookies
    - Always include `Content-Type: application/json` for POST/PUT/PATCH
    - Read CSRF token from `__csrf` cookie and include as `X-CSRF-Token` header on state-changing requests
    - On 401 response: redirect to `/login` (unless already on login page)
    - Throw typed errors with status code and message

    **Auth types (`frontend/src/features/auth/types.ts`):**
    ```typescript
    export interface User {
      id: string;
      username: string;
      isAdmin: boolean;
      totpEnabled: boolean;
    }

    export interface LoginResponse {
      success?: boolean;
      requiresTOTP?: boolean;
      requiresTOTPSetup?: boolean;
      requiresPasswordChange?: boolean;
      error?: string;
    }

    export interface TOTPSetupResponse {
      qrCodeDataURL: string;
    }
    ```

    **Auth API (`frontend/src/features/auth/api.ts`):**
    - `login(username: string, password: string): Promise<LoginResponse>`
    - `verifyTOTP(code: string, rememberDevice?: boolean): Promise<{ success: boolean }>`
    - `setupTOTP(): Promise<TOTPSetupResponse>`
    - `verifyTOTPSetup(code: string): Promise<{ success: boolean }>`
    - `changePassword(newPassword: string, currentPassword?: string): Promise<{ success: boolean }>`
    - `logout(): Promise<void>`
    - `getMe(): Promise<User>`
    All use the apiClient wrapper.

    **Auth hooks (`frontend/src/features/auth/hooks.ts`):**
    - `useAuth()`: TanStack Query hook that calls `getMe()`. Returns `{ user, isLoading, isAuthenticated, isAdmin, refetch }`. Uses `staleTime: 5 * 60 * 1000` (5 min cache).
    - `useLogin()`: TanStack Mutation wrapping `login()`. Returns `{ login, isPending }`. On success, invalidates auth query.
    - `useLogout()`: TanStack Mutation wrapping `logout()`. On success, invalidates auth query and navigates to `/login`.

    **Route protection in `App.tsx`:**
    - Create a `ProtectedRoute` component that wraps `Outlet`:
      - Uses `useAuth()` hook
      - If loading: show full-page skeleton (centered spinner or skeleton layout)
      - If not authenticated: redirect to `/login`
      - If authenticated: render `Outlet` inside AppShell
    - Create a `PublicRoute` component for login page:
      - If already authenticated: redirect to `/`
    - Update router:
      - `/login` wrapped in `PublicRoute`
      - All app routes wrapped in `ProtectedRoute`

    **Header updates (`frontend/src/components/layout/Header.tsx`):**
    - Use `useAuth()` to show actual username and admin badge
    - Use `useLogout()` for logout dropdown item
    - Show first letter of username in avatar
    - If admin, show subtle admin badge next to avatar

    **Login page (`frontend/src/routes/Login.tsx`):**

    Split-screen layout (per locked decision):
    - Left side (50% width): Dark gradient/geometric background with Layer8 logo centered
      - Background: dark gradient from near-black to dark gray, or subtle geometric pattern using CSS (triangles, hexagons, or a repeating grid pattern)
      - Logo: `layer8_logo_dark.jpg` centered vertically and horizontally
      - Full viewport height
    - Right side (50% width): Login form centered vertically
      - White background (light theme) or dark card (dark theme)
      - "Welcome back" heading
      - LoginForm component
      - Below form: subtle "Forgot password?" link (placeholder for future password reset)

    State machine for login flow:
    - `idle` → user sees login form
    - `awaitingTOTP` → user sees TOTP verification dialog
    - `awaitingTOTPSetup` → user sees onboarding wizard
    - `awaitingPasswordChange` → user sees onboarding wizard (starts at password step)

    Transition based on LoginResponse from API.

    **LoginForm (`frontend/src/components/auth/LoginForm.tsx`):**
    - Zod schema: username (required), password (required)
    - React Hook Form with zodResolver
    - Styled with shadcn/ui Input and Button components
    - Generic error toast on failure: "Invalid credentials" (per locked decision — prevents enumeration)
    - Loading state on submit button (spinner icon)
    - On success: call parent callback with response to trigger state transition

    **TOTPVerification (`frontend/src/components/auth/TOTPVerification.tsx`):**
    - For returning users who have TOTP enabled
    - 6-digit code input — use 6 separate single-character inputs that auto-advance (OTP input pattern)
    - OR a single input with `maxLength={6}` and `inputMode="numeric"` (simpler, go with this)
    - "Remember this device for 30 days" checkbox
    - Submit button
    - "Back to login" link to return to username/password
    - Error toast on invalid code
    - Auto-submit when 6 digits are entered

    **TOTPSetup (`frontend/src/components/auth/TOTPSetup.tsx`):**
    - Called during first-login onboarding
    - Displays QR code image from `qrCodeDataURL`
    - Instructions: "Scan this QR code with your authenticator app (Google Authenticator, Authy, etc.)"
    - Manual secret display (hidden by default, toggle to show) for cases where QR scanning doesn't work
    - 6-digit code input to verify setup
    - Submit button: "Verify & Enable MFA"
    - Error handling for invalid code

    **PasswordChange (`frontend/src/components/auth/PasswordChange.tsx`):**
    - For first-login password change
    - Zod schema:
      - `newPassword`: min 8 chars, 1 uppercase, 1 lowercase, 1 number
      - `confirmPassword`: must match newPassword
    - React Hook Form
    - Password strength indicator (optional, Claude's discretion — simple visual bar showing weak/medium/strong based on length + character types)
    - Show/hide password toggle (eye icon)
    - Submit button: "Set Password"

    **OnboardingWizard (`frontend/src/components/auth/OnboardingWizard.tsx`):**
    - Multi-step wizard for first login (per locked decision: guided onboarding)

    - Step indicator at top (numbered circles connected by lines, active step highlighted)
    - Skip step 1 if password change not required
    - "Next" / "Back" navigation between steps
    - Smooth fade transition between steps (per decision: subtle & smooth animations)
    - Card container with generous padding (per decision: spacious & airy)
    - Layer8 logo at top of wizard card

    - Focus states on inputs (ring outline in brand color)
