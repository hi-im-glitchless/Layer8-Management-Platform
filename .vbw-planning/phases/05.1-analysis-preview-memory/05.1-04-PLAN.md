## Objective
Enhance the Python analysis prompt pipeline to accept and inject few-shot examples into LLM Pass 1. The AnalyzeRequest model gains an optional `few_shot_examples` field, and the prompt builder inserts a "## Previous Successful Mappings" section after the reference patterns when examples are provided. This plan is Python-only (sanitization-service) -- the backend KB query that populates these examples is handled in Plan 3. Depends on Plan 1 (KB service defines the data shape that flows through this pipeline).

## Context
- @sanitization-service/app/services/analysis_prompt.py -- build_analysis_prompt() constructs the 5-section prompt. Few-shot section to be inserted between Section 2 (reference patterns) and Section 3 (available GW fields).
- @sanitization-service/app/routes/adapter.py -- POST /analyze endpoint receives AnalyzeRequest and returns prompt. Must accept optional few_shot_examples field.
- @sanitization-service/app/models/adapter.py -- Pydantic models. Add FewShotExample model, update AnalyzeRequest.
- @.vbw-planning/phases/05.1-analysis-preview-memory/05.1-CONTEXT.md -- decisions #12 (3-5 examples ranked by usage_count), #13 (silent few-shot), #11 (inject as "## Previous Successful Mappings" after reference patterns)

### Task 2: Prompt Builder Few-Shot Section
**Commit:** `feat(adapter): add few-shot examples section to analysis prompt builder`
**Files:**
- sanitization-service/app/services/analysis_prompt.py (modified) -- add few-shot section builder

**Details:**
1. Add `_build_few_shot_section(examples: list[FewShotExample]) -> str | None`:
   - Returns None if examples list is empty (no section injected)
   - Section header: "## Previous Successful Mappings\n\nThese section-to-field mappings were confirmed correct in previous template adaptations for this type and language:\n"
   - Each example: `  {i}. Section: "{normalized_section_text}" -> GW Field: {gw_field} [{marker_type}] (confirmed {usage_count} times)`
   - Footer: "\nUse these as reference when mapping similar sections. They represent high-confidence patterns.\n"
2. Import FewShotExample from app.models.adapter
3. Modify `build_analysis_prompt()` signature to accept optional `few_shot_examples: list[FewShotExample] | None = None`
4. Insert few-shot section between Section 2 (reference patterns) and Section 3 (GW fields) when examples are non-empty:
   ```python
   # Between sections.append(_build_reference_patterns_section(...)) and sections.append(_build_gw_fields_section(...)):
   if few_shot_examples:
       few_shot = _build_few_shot_section(few_shot_examples)
       if few_shot:
           sections.append(few_shot)
   ```
5. The few-shot section supplements reference patterns without changing output format or rules
