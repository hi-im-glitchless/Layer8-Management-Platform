## Objective
Redesign the Step 2 (Analysis) frontend to add a tabbed layout with an annotated PDF preview and an enhanced editable mapping table. The Preview tab (default) renders the annotated PDF showing green/yellow paragraph shading with GW field tooltips. The Mappings tab enhances the existing MappingTable with inline editing for gap rows and a paragraph picker for adding missed sections. Both tabs share the same mapping state and sync changes to the backend. This is the final plan and depends on Plans 3 (backend endpoints) and 4 (prompt enhancement).

### Task 2: Hooks for Annotated Preview & Mapping Updates
**Commit:** `feat(adapter-ui): add TanStack Query hooks for annotated preview and inline editing`
**Files:**
- frontend/src/features/adapter/hooks.ts (modified) -- add new hooks

**Details:**
1. Add `useAnnotatedPreview(sessionId: string | null)`:
   - Mutation to request annotated preview (POST)
   - Returns { mutate, data (pdfJobId + metadata), isPending, isError }
2. Add `useAnnotatedPreviewStatus(sessionId: string | null, pdfJobId: string | null)`:
   - Query that polls for PDF readiness using existing preview status endpoint
   - Combines cached tooltipData/unmappedParagraphs from annotated preview response
   - Stops polling when PDF status is 'completed' or 'failed'
3. Add `useUpdateMapping()`:
   - Mutation for POST /api/adapter/update-mapping
   - On success: invalidate session query, update local mapping plan state
   - Returns { mutate, isPending }
4. Add `useCachedAnnotatedPreview(sessionId: string | null)`:
   - Query for GET /api/adapter/annotated-preview/:sessionId
   - Used on page reload to restore annotation metadata without re-generating
