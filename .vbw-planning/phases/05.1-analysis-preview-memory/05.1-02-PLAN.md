## Objective
Build the Python service for annotated DOCX generation and gap detection. The service takes a client DOCX template and a mapping plan, compares mapped fields against the reference template's known GW fields to detect gaps, applies green (mapped) and yellow (gap) paragraph background shading via python-docx, and returns the annotated DOCX plus metadata for the frontend (tooltip data, unmapped paragraphs list). This reuses the existing Gotenberg PDF pipeline from Phase 4 for rendering the annotated DOCX as a PDF.

## Tasks

### Task 1: Gap Detection Service
**Commit:** `feat(adapter): implement gap detection comparing mapping plan against reference template fields`
**Files:**
- sanitization-service/app/services/gap_detector.py (new) -- gap detection logic

**Details:**
1. Create `sanitization-service/app/services/gap_detector.py`
2. Implement `detect_gaps(mapping_plan: MappingPlan, template_type: TemplateType, language: TemplateLanguage) -> GapDetectionResult`:
   - Load reference template via `load_reference_template(template_type, language)`
   - Extract the set of expected GW fields from reference patterns (deduplicated by gw_field)
   - Extract the set of mapped GW fields from mapping_plan.entries (deduplicated by gw_field)
   - Compute missing fields: expected - mapped
   - For each missing field, try to determine expected paragraph position from reference template context
   - Filter out control_flow and table_row_loop fields from gap detection (these are structural, not content)
3. Define Pydantic models:
   - `GapEntry`: gw_field (str), marker_type (str), expected_context (str, from reference pattern), estimated_paragraph_index (int | None)
   - `GapDetectionResult`: gaps (list[GapEntry]), mapped_field_count (int), expected_field_count (int), coverage_percent (float)
4. Coverage percent = mapped_field_count / expected_field_count * 100

### Task 2: Paragraph Shading Service
**Commit:** `feat(adapter): implement DOCX paragraph shading for annotated preview`
**Files:**
- sanitization-service/app/services/annotated_preview.py (new) -- DOCX shading + annotation

**Details:**
1. Create `sanitization-service/app/services/annotated_preview.py`
2. Implement `apply_paragraph_shading(doc_bytes: bytes, mapping_plan: MappingPlan, gaps: list[GapEntry]) -> bytes`:
   - Load DOCX via python-docx Document(BytesIO(doc_bytes))
   - For each MappingEntry in mapping_plan.entries: apply green shading to paragraph at section_index
   - For each GapEntry with estimated_paragraph_index: apply yellow shading to that paragraph
   - Green shading: RGB(198, 239, 206) -- standard "light green" for success
   - Yellow shading: RGB(255, 242, 204) -- standard "light yellow" for warning
   - Apply shading via OOXML paragraph properties: add `<w:shd w:val="clear" w:color="auto" w:fill="C6EFCE"/>` to paragraph's pPr element
   - Use lxml etree manipulation (same pattern as instruction_applier.py)
   - Return modified DOCX bytes
3. Implement helper `_set_paragraph_shading(paragraph, hex_color: str)`:
   - Access paragraph._element (the w:p XML element)
   - Get or create w:pPr child element
   - Get or create w:shd child within pPr
   - Set attributes: val="clear", color="auto", fill=hex_color

**Commit:** `feat(adapter): generate tooltip data and unmapped paragraphs list for frontend`
**Files:**
- sanitization-service/app/services/annotated_preview.py (modified) -- add metadata generation

**Details:**
1. Implement `generate_annotation_metadata(doc_bytes: bytes, mapping_plan: MappingPlan, gaps: list[GapEntry]) -> AnnotationMetadata`:
   - Parse DOCX to get paragraph list
   - Build tooltip_data: array of {paragraph_index, gw_field, marker_type, section_text, status: "mapped"|"gap"} for all mapped + gap paragraphs
   - Build unmapped_paragraphs: array of {paragraph_index, text (first 200 chars), heading_level} for paragraphs NOT in mapping_plan and NOT identified as gaps

   - Sort unmapped by paragraph_index ascending
2. Define Pydantic models:
   - `TooltipEntry`: paragraph_index (int), gw_field (str), marker_type (str), section_text (str), status (Literal["mapped", "gap"])
   - `UnmappedParagraph`: paragraph_index (int), text (str), heading_level (int | None)
   - `AnnotationMetadata`: tooltip_data (list[TooltipEntry]), unmapped_paragraphs (list[UnmappedParagraph])
