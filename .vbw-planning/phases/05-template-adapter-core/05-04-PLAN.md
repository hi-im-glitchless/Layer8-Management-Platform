## Objective
Build the frontend 5-step wizard UI for the Template Adapter feature. The wizard guides the user through uploading a DOCX template, reviewing the LLM analysis, watching adaptation progress, previewing the result, and downloading the adapted template. State is managed client-side with TanStack Query and server-side via the wizard session API. The chat panel uses SSE streaming from the Phase 3 infrastructure. Existing reusable components (FileUpload, PdfPreview) are leveraged.

### Task 1: Feature Module (Types, API, Hooks)
**Commit:** `feat(adapter-ui): add template adapter feature module with types, API, and hooks`
**Files:**
- frontend/src/features/adapter/types.ts -- new
- frontend/src/features/adapter/api.ts -- new
- frontend/src/features/adapter/hooks.ts -- new

**Steps:**
1. Create `frontend/src/features/adapter/types.ts`:
   - `WizardStep`: "upload" | "analysis" | "adaptation" | "preview" | "download"
   - `TemplateType`: "web" | "internal" | "mobile"
   - `TemplateLanguage`: "en" | "pt-pt"
   - `MappingEntry`: sectionIndex, sectionText, gwField, placeholderTemplate, confidence, markerType, rationale
   - `MappingPlan`: entries (MappingEntry[]), templateType, language, warnings (string[])
   - `WizardState`: sessionId, currentStep, templateFile, config, analysis, adaptation, preview, chat
   - `ChatMessage`: role ("user" | "assistant"), content, timestamp
   - `AdaptationProgress`: step (string), status ("pending" | "active" | "complete" | "error"), message (string)
   - `UploadResponse`: sessionId, currentStep
   - `AnalyzeResponse`: mappingPlan, referenceTemplateHash
   - `ApplyResponse`: currentStep, appliedCount, skippedCount, warnings
   - `PreviewResponse`: pdfJobId, docxUrl
2. Create `frontend/src/features/adapter/api.ts`:
   - `adapterApi.uploadTemplate(file: File, templateType, language): Promise<UploadResponse>` -- multipart POST to /api/adapter/upload
   - `adapterApi.analyzeTemplate(sessionId): Promise<AnalyzeResponse>` -- POST to /api/adapter/analyze
   - `adapterApi.applyInstructions(sessionId): Promise<ApplyResponse>` -- POST to /api/adapter/apply
   - `adapterApi.requestPreview(sessionId): Promise<PreviewResponse>` -- POST to /api/adapter/preview
   - `adapterApi.getPreviewStatus(sessionId): Promise<PreviewResponse>` -- GET /api/adapter/preview/:sessionId
   - `adapterApi.downloadUrl(sessionId): string` -- GET /api/adapter/download/:sessionId
   - `adapterApi.getSession(sessionId): Promise<WizardState>` -- GET /api/adapter/session/:sessionId
   - `adapterApi.getActiveSession(): Promise<WizardState | null>` -- GET /api/adapter/session
   - `adapterApi.streamChat(sessionId, message): EventSource` -- POST /api/adapter/chat (SSE)
3. Create `frontend/src/features/adapter/hooks.ts`:
   - `useUploadTemplate()`: useMutation wrapping adapterApi.uploadTemplate
   - `useAnalyzeTemplate()`: useMutation wrapping adapterApi.analyzeTemplate
   - `useApplyInstructions()`: useMutation wrapping adapterApi.applyInstructions
   - `useRequestPreview()`: useMutation wrapping adapterApi.requestPreview
   - `usePreviewStatus(sessionId)`: useQuery with 2s polling (same pattern as usePdfJobStatus)
   - `useWizardSession(sessionId)`: useQuery wrapping adapterApi.getSession
   - `useActiveSession()`: useQuery wrapping adapterApi.getActiveSession
   - `useAdapterChat()`: Custom hook that manages SSE EventSource connection, returns { sendMessage, messages, isStreaming }
     - Opens POST request to /api/adapter/chat
     - Listens for delta, mapping_update, done, error events
     - Accumulates messages in state
     - Handles disconnect and reconnection
