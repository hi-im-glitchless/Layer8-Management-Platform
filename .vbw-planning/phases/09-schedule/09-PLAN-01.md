## Task 1: Add Prisma models for Schedule feature

**Commit:** `feat(09): add Prisma models for schedule, assignments, holidays, absences`

**File:** `backend/prisma/schema.prisma`

Append these models at the end of `schema.prisma`:

```prisma
model SchedulePerson {
  id          String   @id @default(cuid())
  name        String
  sortOrder   Int      @default(0)
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  assignments ScheduleAssignment[]
  absences    Absence[]

  @@index([sortOrder])
}

model ScheduleAssignment {
  id          String   @id @default(cuid())
  personId    String
  weekStart   String   // ISO date string "YYYY-MM-DD" (always a Monday)
  year        Int
  projectName String   @default("")
  color       String   @default("#3b82f6") // hex color from palette
  status      String   @default("confirmed") // confirmed|needs-reqs|placeholder
  isLocked    Boolean  @default(false)
  splitIndex  Int      @default(0) // 0 = full cell or first half, 1 = second half
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  person SchedulePerson @relation(fields: [personId], references: [id], onDelete: Cascade)

  @@unique([personId, weekStart, splitIndex])
  @@index([personId])
  @@index([year])
  @@index([weekStart])
}

model Holiday {
  id    String @id @default(cuid())
  name  String
  date  String // ISO date string "YYYY-MM-DD"
  year  Int

  @@unique([date])
  @@index([year])
}

model Absence {
  id       String @id @default(cuid())
  personId String
  date     String // ISO date string "YYYY-MM-DD"

  person SchedulePerson @relation(fields: [personId], references: [id], onDelete: Cascade)

  @@unique([personId, date])
  @@index([personId])
  @@index([date])
}
```

After editing, run:
```bash
cd backend && npx prisma migrate dev --name add-schedule-models
```

## Task 3: Create schedule routes with RBAC guards

**Commit:** `feat(09): add schedule Express routes with RBAC and Zod validation`

**File:** `backend/src/routes/schedule.ts` (NEW)

Create an Express Router following the pattern in `routes/users.ts`:

```typescript
import { Router } from 'express';
import { z } from 'zod';
import { requireRole } from '../middleware/auth.js';
import { auditMiddleware } from '../middleware/audit.js';
import * as scheduleService from '../services/schedule.js';

const router = Router();

// All routes require at least NORMAL (authenticated)
// Write operations require MANAGER

// --- Persons ---
// GET /api/schedule/persons — requireRole('NORMAL')
// POST /api/schedule/persons — requireRole('MANAGER'), auditMiddleware('schedule.person.create')
//   Body: { name: string }
// PUT /api/schedule/persons/:id — requireRole('MANAGER'), auditMiddleware('schedule.person.update')
//   Body: { name?: string, sortOrder?: number, isActive?: boolean }
// DELETE /api/schedule/persons/:id — requireRole('MANAGER'), auditMiddleware('schedule.person.delete')
// PUT /api/schedule/persons/reorder — requireRole('MANAGER'), auditMiddleware('schedule.person.reorder')
//   Body: { orderedIds: string[] }

// --- Assignments ---
// GET /api/schedule/assignments?year=2026 — requireRole('NORMAL')
// PUT /api/schedule/assignments — requireRole('MANAGER'), auditMiddleware('schedule.assignment.upsert')
//   Body: { personId, weekStart, year, projectName, color, status, isLocked, splitIndex }
// DELETE /api/schedule/assignments/:id — requireRole('MANAGER'), auditMiddleware('schedule.assignment.delete')
// POST /api/schedule/assignments/swap — requireRole('MANAGER'), auditMiddleware('schedule.assignment.swap')
//   Body: { fromId: string, toPersonId: string, toWeekStart: string }

// --- Holidays ---
// GET /api/schedule/holidays?year=2026 — requireRole('NORMAL')
// PUT /api/schedule/holidays — requireRole('MANAGER'), auditMiddleware('schedule.holidays.set')
//   Body: { year: number, holidays: { name: string, date: string }[] }

// --- Absences ---
// GET /api/schedule/absences?year=2026&personId=xxx — requireRole('NORMAL')
// POST /api/schedule/absences/toggle — requireRole('MANAGER'), auditMiddleware('schedule.absence.toggle')
//   Body: { personId: string, date: string }

export default router;
```

Each route handler:
1. Defines a Zod schema inline (like `users.ts` pattern)
2. Parses request body/query
3. Delegates to `scheduleService.*`
4. Returns JSON response
5. Catches ZodError → 400, other errors → 500

**IMPORTANT:** The reorder endpoint MUST be registered BEFORE the `:id` param routes to avoid Express treating "reorder" as an id.

## Task 5: Seed Portuguese public holidays for 2026

**Commit:** `feat(09): add seed script for 2026 Portuguese public holidays`

**File:** `backend/prisma/seed-holidays.ts` (NEW)

Create a seed script that inserts the standard Portuguese public holidays for 2026:

```typescript
import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

const PORTUGUESE_HOLIDAYS_2026 = [
  { name: 'Ano Novo', date: '2026-01-01' },
  { name: 'Sexta-feira Santa', date: '2026-04-03' },
  { name: 'Domingo de Páscoa', date: '2026-04-05' },
  { name: 'Dia da Liberdade', date: '2026-04-25' },
  { name: 'Dia do Trabalhador', date: '2026-05-01' },
  { name: 'Corpo de Deus', date: '2026-06-04' },
  { name: 'Dia de Portugal', date: '2026-06-10' },
  { name: 'Assunção de Nossa Senhora', date: '2026-08-15' },
  { name: 'Implantação da República', date: '2026-10-05' },
  { name: 'Dia de Todos os Santos', date: '2026-11-01' },
  { name: 'Restauração da Independência', date: '2026-12-01' },
  { name: 'Imaculada Conceição', date: '2026-12-08' },
  { name: 'Natal', date: '2026-12-25' },
];

async function main() {
  const year = 2026;
  // Delete existing holidays for the year
  await prisma.holiday.deleteMany({ where: { year } });
  // Insert all holidays
  await prisma.holiday.createMany({
    data: PORTUGUESE_HOLIDAYS_2026.map(h => ({ ...h, year })),
  });
  console.log(`Seeded ${PORTUGUESE_HOLIDAYS_2026.length} holidays for ${year}`);
}

main()
  .catch(console.error)
  .finally(() => prisma.$disconnect());
```

Run with: `cd backend && npx tsx prisma/seed-holidays.ts`
