# Plan 04: Edit Modal, Team Panel, and Holiday Configuration

## Overview

Build the management UI components: the assignment edit modal (click cell to edit), the team management side panel (add/remove/reorder members), and the holiday configuration dialog. These are rendered as overlays and do not modify the grid component files from Plan 03.

---

## Task 1: Create AssignmentEditModal component

**Commit:** `feat(09): add AssignmentEditModal with color palette and status controls`

**File:** `frontend/src/features/schedule/components/AssignmentEditModal.tsx` (NEW)

A modal dialog for editing a single assignment cell:

**Props:**
```typescript
interface AssignmentEditModalProps {
  open: boolean
  onOpenChange: (open: boolean) => void
  personId: string
  personName: string
  weekStart: string         // "YYYY-MM-DD"
  year: number
  splitIndex: number
  existingAssignment?: ScheduleAssignment  // pre-fill if editing
  onSave: (data: UpsertAssignmentRequest) => void
  onDelete: () => void
}
```

**UI (shadcn components):**
- `Dialog` + `DialogContent` + `DialogHeader` + `DialogTitle` + `DialogFooter`
- Title: "{personName} — Week of {formatted weekStart}"
- Form fields (React Hook Form + Zod):
  - **Project Name**: `Input` component, required, placeholder "Project name"
  - **Color**: Grid of 20 color swatches (from `PROJECT_COLORS`), each a 28px circle button. Selected one gets a ring. Arranged in 5x4 grid.
  - **Status**: 3 radio-style buttons cycling through `ASSIGNMENT_STATUSES`. Each shows icon + label. Use `ToggleGroup` or custom buttons.
  - **Locked**: `Switch` component with "Lock assignment" label
  - **Split**: Info text "This is the {first|second} half of a split week" if splitIndex > 0 or if another assignment exists for the other split

**Footer buttons:**
- "Delete" (destructive, left-aligned) — only shown if editing existing assignment
- "Cancel" (outline)
- "Save" (primary)

**Behavior:**
- On Save: call `onSave` with the form data
- On Delete: call `onDelete` after confirmation
- Close modal on successful save

**UI:**
- Use shadcn `Sheet` + `SheetContent` (side="right") + `SheetHeader` + `SheetTitle`

## Task 3: Create HolidayConfigDialog component

**Commit:** `feat(09): add HolidayConfigDialog for managing public holidays`

**File:** `frontend/src/features/schedule/components/HolidayConfigDialog.tsx` (NEW)

A dialog for viewing and editing the holiday list for the current year:

**Props:**
```typescript
interface HolidayConfigDialogProps {
  open: boolean
  onOpenChange: (open: boolean) => void
  year: number
}
```

**UI:**
- `Dialog` + `DialogContent` (max-width 500px)
- Title: "{year} Public Holidays"
- **Holiday list**: each row shows:
  - Date (formatted as "dd MMM" e.g. "25 Dec") — using `date-fns` `format`
  - Name (editable `Input`)
  - Day of week label (e.g. "Thursday") — computed from date
  - Delete button (`X` icon)
- **Add row** at bottom: date picker (`Input type="date"`) + name `Input` + "Add" `Button`
- **Save button** in footer: calls `useSetHolidays` with the full list

**Behavior:**
- Load initial data from `useHolidays(year)` hook
- Maintain local state (cloned from server data) for editing
- Save sends the entire list as a replacement
- Sort by date ascending in display
