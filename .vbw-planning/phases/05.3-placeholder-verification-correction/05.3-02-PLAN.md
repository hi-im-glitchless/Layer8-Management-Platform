## Objective
Restructure the frontend wizard from 5 steps to 4 steps: Upload -> Verify Placeholders -> Preview -> Download. The Upload step gains inline progress display for auto-mapping (LLM analysis + insertion runs automatically after upload). The old Analysis and Adaptation steps are removed. This plan implements Decisions #10 (4-step wizard), #11 (progress in Upload step, auto-advance to Verify), and prepares the routing for Plan 03's StepVerify component. Does not yet create StepVerify -- that is Plan 03.

## Context
- @frontend/src/features/adapter/types.ts -- WizardStep type, WizardState interface. Must change to 4-step model. Also contains all SSE event types, selection types, etc. that remain unchanged.
- @frontend/src/features/adapter/components/StepIndicator.tsx -- STEPS array and STEP_ORDER record. Currently 5 steps. Exported constants used by WizardShell.
- @frontend/src/features/adapter/components/WizardShell.tsx -- Main container with STEP_SEQUENCE, lazy imports, renderStep() switch, navigation buttons. Must remove 'analysis'/'adaptation' cases, add 'verify'.
- @frontend/src/features/adapter/components/StepUpload.tsx -- Currently just uploads and calls onSessionCreate. Needs to also trigger auto-map and show progress.
- @frontend/src/features/adapter/components/AnalysisProgress.tsx -- Existing progress stepper. Reuse for auto-map progress in Upload step with updated step labels.
- @frontend/src/features/adapter/hooks.ts -- Need new useAutoMap() hook for POST /auto-map endpoint.
- @frontend/src/features/adapter/api.ts -- Need new autoMap() API function.
- Decision #10: 4 steps: Upload -> Verify -> Preview -> Download
- Decision #11: Progress steps in Upload step, auto-advance to Verify. Reuse existing progress infra with persistence fixes.
- Decision #16: Translation (Phase 6) inserts as optional Step 4 between Preview and Download (future -- design type system to support this)

### Task 4: Auto-Map Hook & Upload Step Progress
**File(s):** `frontend/src/features/adapter/hooks.ts`, `frontend/src/features/adapter/components/StepUpload.tsx`
**Action:** Modify
**Details:**
- In hooks.ts, add useAutoMap() mutation hook:
  ```typescript
  export function useAutoMap() {
    const queryClient = useQueryClient()
    return useMutation({
      mutationFn: (sessionId: string) => adapterApi.autoMap(sessionId),
      onSuccess: (_data, sessionId) => {
        queryClient.invalidateQueries({ queryKey: ['adapter', 'session', sessionId] })
      },
      onError: (error: Error) => {
        toast.error(error.message || 'Auto-mapping failed')
      },
    })
  }
  ```
- In StepUpload.tsx, extend the component to handle auto-map after upload:
  - Add props: `onAutoMapComplete: (sessionId: string) => void` (replaces onSessionCreate for the full flow)
  - After upload success, automatically trigger autoMap mutation with the returned sessionId
  - While auto-map runs, show AnalysisProgress component inline below the upload form (reuse from AnalysisProgress.tsx)
  - Track elapsed time with sessionStorage persistence (same pattern as StepAnalysis)
  - Update progress steps for auto-map: [1] "Analyzing template structure...", [2] "LLM mapping placeholders...", [3] "Applying placeholders to document...", [4] "Generating placeholder preview..."
  - On auto-map success: call onAutoMapComplete(sessionId), show toast "Template auto-mapped successfully"
  - On auto-map error: show error in AnalysisProgress with retry button
  - Disable the upload form while auto-map is running
  - Session polling fallback: if auto-map takes >60s, poll session every 10s to detect server-side completion (same pattern as StepAnalysis)
**Commit:** `feat(adapter-ui): add auto-map hook and inline progress to StepUpload`
