## What Was Built
- New StepVerify component replacing StepAnalysis as the 'verify' step in the 4-step wizard
- Backend proxy POST /api/adapter/placeholder-preview: reads adapted DOCX, calls Python service, saves annotated DOCX, queues PDF conversion
- generatePlaceholderPreview() service function with PlaceholderInfo/PlaceholderPreviewResult types
- Frontend requestPlaceholderPreview() API and usePlaceholderPreview() mutation hook
- PlaceholderInfo and PlaceholderPreviewResponse frontend types
- StepVerify renders placeholder-styled PDF on left, always-visible chat panel on right (grid layout)
- Selection state wired: text select on PDF creates numbered badges, chat references #N for corrections
- SSE event handling for selection_mapping and batch_complete correction events
- Coverage counter toolbar badge showing "X placeholders" count, updates after regeneration
- Approve button advances to Preview step; disabled until PDF loads
- Refresh Preview button appears when mapping corrections make preview outdated (Decision #8: clears selections)
- KB badge with tooltip for knowledge base persistence indicator
- WizardShell routes 'verify' case to StepVerify, StepPreview onReAdapt goes to 'verify'
