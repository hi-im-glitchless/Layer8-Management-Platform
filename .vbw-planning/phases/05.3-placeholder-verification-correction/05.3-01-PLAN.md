## Objective
Build the backend infrastructure for Phase 5.3's auto-map-on-upload and placeholder-styled preview. After DOCX upload, the system automatically runs LLM Pass 1 (analysis) + Pass 2 (insertion) without user intervention, then generates a PDF showing the DOCX with visible Jinja placeholders styled with light blue backgrounds. This plan implements Decisions #1 (auto-map on upload), #2 (styled placeholders with light background), and #10 (merge Analysis + Adaptation into auto-map). The frontend will consume these endpoints in Plans 02 and 03.

### Task 2: Auto-Map Service Function
**File(s):** `backend/src/services/templateAdapter.ts`
**Action:** Modify
**Details:**
- Add new exported function `autoMapTemplate(wizardState: WizardState): Promise<WizardState>`:
  1. Read templateBase64 from wizardState.templateFile.base64
  2. Call `analyzeTemplate(templateBase64, config.templateType, config.language)` -- Pass 1
  3. Update wizard state with analysis results (mappingPlan, referenceTemplateHash)
  4. Call `applyInstructions(updatedWizardState)` -- Pass 2 (builds insertion prompt, LLM generates instructions, applies to DOCX)
  5. The applyInstructions() call saves the adapted DOCX and updates wizard state
  6. Update wizard state final step to 'verify' (not 'adaptation')
  7. Return the final wizard state
- Error handling: if Pass 1 fails, throw with context. If Pass 2 fails, preserve the analysis results in state and throw (the mapping plan is still usable for retry)
- This function replaces the separate analyze -> apply flow for the 5.3 wizard
**Commit:** `feat(adapter): add autoMapTemplate service that chains Pass 1 + Pass 2`

### Task 4: Placeholder Preview Service (Python)
**File(s):** `sanitization-service/app/services/annotated_preview.py`, `sanitization-service/app/models/adapter.py`
**Action:** Modify
**Details:**
- Add new constant: `PLACEHOLDER_SHADING = "DAEAF6"` (light blue, RGB(218,234,246))
- Add new model in adapter.py:
  ```python
  class PlaceholderPreviewRequest(BaseModel):
      adapted_base64: str  # Base64 DOCX WITH Jinja placeholders inserted
      template_type: str
      language: str

  class PlaceholderInfo(BaseModel):
      paragraph_index: int
      placeholder_text: str  # e.g., "{{ client.short_name }}"
      gw_field: str  # extracted field path

  class PlaceholderPreviewResponse(BaseModel):
      annotated_base64: str
      placeholders: list[PlaceholderInfo]
      placeholder_count: int
  ```
- Add new function `generate_placeholder_preview(doc_bytes: bytes) -> tuple[bytes, list[PlaceholderInfo]]`:
  1. Open DOCX with python-docx
  2. Scan all paragraphs for Jinja2 expressions using regex: `r'\{\{.*?\}\}'`
  3. For each paragraph containing at least one placeholder:
     - Apply light blue shading via `_set_paragraph_shading(paragraph, PLACEHOLDER_SHADING)`
     - Extract each placeholder expression and its field path (strip `{{ }}` and whitespace)
     - Record PlaceholderInfo(paragraph_index, placeholder_text, gw_field)
  4. Save modified DOCX to bytes
  5. Return (annotated_bytes, placeholder_list)
- Also scan tables: iterate doc.tables -> rows -> cells -> paragraphs for placeholders
**Commit:** `feat(adapter): add placeholder preview service with light blue Jinja shading`
