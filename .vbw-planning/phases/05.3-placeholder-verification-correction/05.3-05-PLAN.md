### Task 2: PDF Scroll-to-Placeholder
**File(s):** `frontend/src/features/adapter/components/InteractivePdfViewer.tsx`, `frontend/src/features/adapter/components/StepVerify.tsx`
**Action:** Modify
**Details:**
- In InteractivePdfViewer.tsx, expose scroll functionality:
  - Add `onScrollToPage` prop: `(pageNumber: number) => void` callback that the parent can trigger
  - Actually, better approach: add `scrollTargetPage` prop (number | null). When it changes to a non-null value, scroll the PDF container to that page.
  - Implementation: when scrollTargetPage changes, find the `.react-pdf__Page[data-page-number="${n}"]` element and scrollIntoView({ behavior: 'smooth', block: 'start' })
  - After scrolling, the parent resets scrollTargetPage to null
- In StepVerify.tsx, wire PlaceholderNavigator:
  - Import and render PlaceholderNavigator alongside the toolbar
  - onJumpToPlaceholder: estimate which PDF page the paragraph is on (simple heuristic: paragraphIndex / totalParagraphs * totalPages, or use the paragraphs-per-page data from document structure if available)
  - Set scrollTargetPage state to trigger the scroll
  - Toggle state managed locally (same as structureBrowserOpen in StepAnalysis)
**Commit:** `feat(adapter-ui): add scroll-to-placeholder support in InteractivePdfViewer`
