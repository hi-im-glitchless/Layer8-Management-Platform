## What Was Built
- Python correction prompt builder: build_correction_system_prompt() and build_correction_user_prompt() produce structured LLM prompts from current mapping plan + user corrections + selection context
- POST /adapter/build-correction-prompt endpoint with CorrectionPromptRequest/Response models
- Backend processCorrectionChat() async generator: correction prompt -> LLM stream -> validate-mapping -> update state -> applyInstructions() -> generatePlaceholderPreview() -> SSE events
- processChatFeedback() routes to correction mode when currentStep is 'verify' and message has #N refs
- SSE events: correction_result (updated mapping plan) and regeneration_complete (new pdfJobId + count)
- Frontend useAdapterChat() tracks latestCorrectionResult and regenerationResult with clearCorrectionState()
- StepVerify watches correction events, updates PDF viewer, clears selections, shows regeneration spinner overlay
