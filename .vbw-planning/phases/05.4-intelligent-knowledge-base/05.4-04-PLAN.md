## Objective
Replace the correction chat UI with a direct editable mapping table. Users see all auto-mapped entries pre-filled in a table, can edit GW fields via autocomplete, delete rows to remove mappings, and add new rows by highlighting text on the PDF. Bidirectional table-PDF sync replaces the structure browser as the navigation tool. This addresses acceptance criteria #10, #11, #12, #13 and Decision #13 (chat lost 70% of corrections).

### Task 1: GW Field Autocomplete Component
**File(s):** `frontend/src/features/adapter/components/GwFieldAutocomplete.tsx` (new file)
**Action:** Create
**Details:**
- Create autocomplete/typeahead component for GW field selection:
  ```typescript
  interface GwFieldAutocompleteProps {
    value: string
    onChange: (gwField: string, markerType: string, jinja2: string) => void
    onRawInput?: (raw: string) => void
    placeholder?: string
    autoFocus?: boolean
  }
  ```
- Internal state: input text, dropdown open, filtered options
- GW field data: hardcode the FIELD_MARKER_MAP entries as a constant array (mirrors Python FIELD_MARKER_MAP). Each entry: `{ gwField, markerType, description, jinja2Template }`. The jinja2Template auto-generates the correct Jinja2 syntax:
  - text fields: `{{ client.short_name }}`
  - paragraph_rt fields: `{{p finding.description_rt }}`
  - run_rt fields: `{{r finding.severity_rt }}`
  - table_row_loop: `{%tr for item in scope %}`
  - control_flow: `{% if ... %}`
- Dropdown filters by input text (fuzzy match on gwField and description)
- Selecting from dropdown calls `onChange(gwField, markerType, jinja2Template)`
- Typing a raw string and pressing Enter calls `onRawInput(raw)` -- supports arbitrary Jinja2 for advanced users / bootstrapping
- Keyboard navigation: Arrow Up/Down to navigate, Enter to select, Escape to close
- Styled with existing shadcn/ui Input + Popover + Command pattern
**Commit:** `feat(adapter-ui): create GwFieldAutocomplete component with typeahead and Jinja2 generation`
