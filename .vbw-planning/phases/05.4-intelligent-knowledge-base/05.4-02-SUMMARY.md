## What Was Built

Zone-aware DOCX parsing and heuristic blueprint detection for the Knowledge Base. The parser now tags every paragraph with its document zone (header, footer, body, table_cell, cover). A new blueprint detector module analyzes completed mapping plans to find loop, group, and conditional patterns without LLM calls. Style hint collection counts mapped vs skipped paragraphs per style/zone for prompt noise reduction. A POST /adapter/detect-blueprints endpoint exposes both capabilities.

## Tasks Completed

1. **Zone Field on DocxParagraph Model** (1c6a464) -- Added ZoneType literal, zone and table_index fields to DocxParagraph with None defaults for backward compat
2. **Zone Tagging in DocxParserService** (de10cf7) -- Body paragraphs tagged body/cover, header/footer paragraphs tagged accordingly, table cell paragraphs tagged table_cell with table_index
3. **Blueprint Detection Heuristics** (dd0cf8b) -- Created blueprint_detector.py with detect_blueprints() implementing loop, group, and conditional pattern detection
4. **Style Hint Collection** (5bed14c) -- Added collect_style_hints() counting mapped vs skipped paragraphs per (style, zone) combination
