## Objective
Replace the flat few-shot prompt section with a structured KB context block (zone distribution, blueprints, confidence notes). Wire zone data through the full persistence path. Implement cross-type fallback for first-use template types. Add style-based boilerplate filtering to the analysis prompt. This plan addresses acceptance criteria #5, #7, #8 and significantly improves auto-map accuracy.

This gets replaced with structured block. _build_doc_structure_section() at line 125-153 emits numbered paragraphs -- needs boilerplate filtering.

**Details:**

### Task 2: Structured KB Context Block (Python Prompt Builder)
**File(s):** `sanitization-service/app/services/analysis_prompt.py`, `sanitization-service/app/models/adapter.py`
**Action:** Modify
**Details:**
- Add new model `KBContext` in adapter.py:
  ```python
  class KBContextMapping(BaseModel):
      normalized_section_text: str
      gw_field: str
      marker_type: str
      confidence: float
      zone: str = "unknown"
      zone_repetition_count: int = 1

  class BlueprintContext(BaseModel):
      zone: str
      pattern_type: str
      markers: list[dict]
      anchor_style: str | None = None

  class KBContext(BaseModel):
      zone_mappings: dict[str, list[KBContextMapping]] = {}
      blueprints: list[BlueprintContext] = []
      boilerplate_styles: list[str] = []
      repetition_summary: list[dict] = []
      is_cross_type_fallback: bool = False
  ```
- Update AnalyzeRequest to accept `kb_context: KBContext | None = None` alongside existing `few_shot_examples` (backward compat)
- Replace `_build_few_shot_section()` with `_build_kb_context_block(kb_context: KBContext) -> str | None`:
  - **Zone Distribution** subsection: for each zone with mappings, list "Zone: {zone} ({N} patterns)" followed by top-5 entries per zone sorted by confidence desc. Include repetition hints like "client.short_name appears 8x in body, 42x in tables"
  - **Blueprints** subsection: for each blueprint, describe the pattern "Loop: {for finding in findings} -> [{marker1}, {marker2}, ...] in zone {zone}"
  - **Confidence Notes** subsection: list any entries with confidence < 0.5 as "Low confidence: {text} -> {field} ({confidence})" so LLM knows to double-check these
  - If is_cross_type_fallback, add note: "NOTE: These patterns are from other template types (0.7x confidence penalty applied). Adapt with caution."
- Update `build_analysis_prompt()` to call `_build_kb_context_block()` when kb_context is present, falling back to `_build_few_shot_section()` for backward compatibility
**Commit:** `feat(prompt): replace flat few-shot with structured KB context block (zones + blueprints + confidence)`

### Task 3: Token Budget Enforcement
**File(s):** `sanitization-service/app/services/analysis_prompt.py`
**Action:** Modify
**Details:**
- Add helper function `_estimate_tokens(text: str) -> int`: returns `len(text.split()) / 0.75` as rough token estimate
- Add function `_enforce_token_budget(kb_block: str, total_prompt: str, min_pct: float = 0.15, max_pct: float = 0.25, hard_cap: int = 1500) -> str`:
  1. Estimate total prompt tokens (without KB block)
  2. Calculate dynamic budget: `max(min_pct * total_tokens, min(max_pct * total_tokens, hard_cap))`
  3. If KB block exceeds budget, progressively trim:
     - First: remove Confidence Notes section
     - Second: reduce entries per zone from 5 to 3
     - Third: remove cross-type fallback entries
     - Fourth: truncate to budget by removing lowest-confidence entries
  4. Return trimmed KB block
- Wire into `build_analysis_prompt()`: after building all sections, apply budget to KB block before joining
**Commit:** `feat(prompt): enforce dynamic 15-25% token budget on KB context block`
