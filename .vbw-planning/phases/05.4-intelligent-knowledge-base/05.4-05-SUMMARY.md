## What Was Built

Correction trigger system that immediately updates the knowledge base when users edit mappings in the table. The POST /correction-update endpoint decays old mappings (0.7x confidence) and creates/boosts corrected entries. Download persistence was upgraded from bulk-confirm to smart mode detection: compares each entry against existing KB data to determine if it is a confirmation, correction, or new entry. The frontend wires table edits through handleMappingPlanChange which diffs against the original auto-map plan and fires corrections to the backend. A GET /kb-stats endpoint returns zone distribution, blueprint count, style hint count, average confidence, and top fields for the KB badge tooltip. The regeneration flow was polished: Regenerate button saves mappings to wizard state, re-applies the DOCX, generates a new placeholder PDF, and polls for completion with proper error handling.
