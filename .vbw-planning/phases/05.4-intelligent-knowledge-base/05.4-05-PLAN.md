### Task 1: Correction Update Backend Endpoint
**File(s):** `backend/src/routes/templateAdapter.ts`, `backend/src/services/templateAdapter.ts`
**Action:** Modify
**Details:**
- Add new `POST /api/adapter/correction-update` route:
  ```typescript
  // Zod schema
  const correctionUpdateSchema = z.object({
    sessionId: z.string(),
    corrections: z.array(z.object({
      sectionIndex: z.number(),
      oldGwField: z.string(),
      newGwField: z.string(),
      newMarkerType: z.string(),
      sectionText: z.string(),
    })),
  })
  ```
  - requireAuth middleware
  - For each correction entry:
    1. If oldGwField differs from newGwField: decay old mapping (mode='correct'), create/boost new mapping (mode='create' or 'confirm')
    2. If only markerType changed: update existing entry
  - Get templateType + language from wizard session
  - Return `{ updated: number, decayed: number }`
- Add service function `processCorrectionUpdate()` in templateAdapter.ts:
  1. Fetch wizard state for templateType/language
  2. For each correction, call upsertMapping with appropriate mode
  3. Log correction count for debugging
- Non-blocking: errors logged but not thrown to caller
**Commit:** `feat(adapter): add POST /correction-update endpoint for table-based KB corrections`

### Task 2: Smart Download Persistence
**File(s):** `backend/src/services/templateAdapter.ts`
**Action:** Modify
**Details:**
- Update `persistMappingsToKB()` to differentiate confirmations from corrections:
  1. Fetch existing KB entries for this (templateType, language) via queryByZone()
  2. Build a lookup map: `Map<string, TemplateMapping>` keyed by normalizedSectionText+gwField
  3. For each entry in the mapping plan:
     - If an existing KB entry with same normalizedSectionText maps to a DIFFERENT gwField: this is a correction. Call upsertMapping(oldEntry, mode='correct') then upsertMapping(newEntry, mode='create')
     - If existing entry matches: this is a confirmation. Call upsertMapping(entry, mode='confirm')
     - If no existing entry: this is new. Call upsertMapping(entry, mode='create')
  4. Include zone data from doc structure (derive zone from paragraph position if available in wizard state)
- Keep the fire-and-forget pattern: errors logged, never thrown
- After mapping persistence, trigger blueprint detection (from Plan 03 Task 5) and style hint storage
**Commit:** `feat(adapter): smart download persistence with confirmation vs correction mode detection`
