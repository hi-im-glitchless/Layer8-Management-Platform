## Tasks

### Task 1: Prisma Schema Evolution (TemplateMapping + New Models)
**File(s):** `backend/prisma/schema.prisma`
**Action:** Modify
**Details:**
- Add three new fields to TemplateMapping model:
  - `zone String @default("unknown")` -- header|footer|body|table_cell|text_box|cover|unknown
  - `zoneRepetitionCount Int @default(1)` -- how many times this gwField appears in this zone
  - `correctionCount Int @default(0)` -- number of times user corrected this mapping
- Add new index: `@@index([templateType, language, zone])` for zone-filtered queries
- Add BlueprintPattern model:
  ```prisma
  model BlueprintPattern {
    id           String   @id @default(cuid())
    templateType String
    zone         String
    patternType  String   // loop|conditional|group
    markers      String   // JSON array: [{"gwField":"...","markerType":"..."},...]
    anchorStyle  String?  // paragraph style that anchors this pattern
    createdAt    DateTime @default(now())
    updatedAt    DateTime @updatedAt
    @@unique([templateType, zone, patternType, markers])
    @@index([templateType])
  }
  ```
- Add StyleHint model:
  ```prisma
  model StyleHint {
    id           String   @id @default(cuid())
    templateType String
    styleName    String
    zone         String   @default("body")
    mappedCount  Int      @default(0)
    skippedCount Int      @default(0)
    createdAt    DateTime @default(now())
    updatedAt    DateTime @updatedAt
    @@unique([templateType, styleName, zone])
    @@index([templateType])
  }
  ```
- Run `npx prisma migrate dev --name add-kb-intelligence` to create migration
**Commit:** `feat(db): add zone fields to TemplateMapping, BlueprintPattern and StyleHint models`

### Task 2: Confidence Decay/Boost in Upsert Functions
**File(s):** `backend/src/services/templateMapping.ts`
**Action:** Modify
**Details:**
- Add new type `UpsertMode = 'confirm' | 'correct' | 'create'` to control confidence behavior
- Extend `templateMappingSchema` with optional fields: `zone`, `zoneRepetitionCount`
- Update `upsertMapping()` signature to accept `mode: UpsertMode = 'confirm'`:
  - `'confirm'`: increment usageCount, boost confidence by `Math.min(1.0, existing + 0.1)`
  - `'correct'`: increment correctionCount, decay confidence by `existing * 0.7`
  - `'create'`: set initial values (usageCount=1, confidence from input)
- Update `bulkUpsertMappings()` to accept `mode` parameter and pass it through
- For `'confirm'` and `'correct'` modes, fetch existing record first to compute new confidence (cannot do arithmetic in Prisma upsert update clause with existing values otherwise)
- Add `zone` and `zoneRepetitionCount` to create/update clauses
**Commit:** `feat(kb): implement confidence decay/boost modes in upsert functions`

### Task 3: Zone-Aware Query Functions

**Action:** Modify
**Details:**
- Update `queryFewShotExamples()` to accept optional `zone?: string` parameter and filter by zone when provided
- Change ordering from `usageCount desc` to `confidence desc, usageCount desc` (confidence is now the primary signal)
- Add new `queryByZone()` function that returns results grouped by zone:
  ```typescript
  export async function queryByZone(
    templateType: string,
    language: string,
    minConfidence: number = 0.3,
  ): Promise<Map<string, TemplateMapping[]>>
  ```
  - Queries all mappings for (templateType, language) where confidence >= minConfidence
  - Groups results by zone field into a Map
  - Orders within each zone by confidence desc
- Add new `queryZoneRepetitionSummary()` function:
  ```typescript

    templateType: string,
    language: string,
  ): Promise<Array<{ gwField: string; zone: string; totalCount: number }>>
  ```
  - Returns aggregated zoneRepetitionCount per (gwField, zone) for the template type
  - Used by the structured prompt builder in Plan 05.4-03
**Commit:** `feat(kb): add zone-aware query and repetition summary functions`
