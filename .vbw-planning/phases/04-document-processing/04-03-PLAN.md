## Context
Build the Ghostwriter GraphQL client in the Node.js backend. This service fetches report data from Ghostwriter's Hasura-based GraphQL API and transforms it into the Jinja2 template context format expected by the reference templates. The GW env vars (URL, token, report ID) are already defined in backend config.ts as optional fields. The data mapping was determined by cross-referencing live GW schema introspection with Jinja2 placeholders extracted from the 8 reference templates.

   ```typescript
   }
   ```
2. Create `backend/src/services/ghostwriter.ts`:
   - `fetchReportData(reportId: number): Promise<GWReport>` -- Sends GraphQL query to `${config.GHOSTWRITER_URL}/v1/graphql` with `Authorization: Bearer ${config.GHOSTWRITER_API_TOKEN}` header
   - Uses raw `fetch()` (no GraphQL library needed for a single query pattern)
   - TLS: Use `NODE_TLS_REJECT_UNAUTHORIZED=0` in dev, or Node.js custom agent with `rejectUnauthorized: false` for the GW fetch only (self-signed cert)
   - GraphQL query fetches: report (id, title, creation), project (codename, dates, client, scopes, assignments with user/role), findings ordered by position (all fields including severity, findingType, CVSS, rich text fields)
   - Transform snake_case GW response to camelCase TypeScript interfaces
   - `checkGhostwriterHealth(): Promise<{ available: boolean; username?: string }>` -- Sends `whoami` query to verify connection and auth
   - Error handling: connection refused, auth failure (401/403), report not found, GraphQL errors
3. Update `backend/src/config.ts`: Keep GW fields optional (not all dev environments will have GW running). Validate URL format when present.

### Task 2: GW Data to Jinja2 Context Transformer
**Commit:** `feat(ghostwriter): implement GW data to Jinja2 template context transformer`
**Files:**
- backend/src/services/ghostwriterMapper.ts -- new file

**Steps:**
1. Create `backend/src/services/ghostwriterMapper.ts`:
   - `mapReportToTemplateContext(report: GWReport): TemplateContext` -- Transforms GW report data to the Jinja2 context format expected by reference templates
   - Output interface `TemplateContext`:
     ```typescript
     export interface TemplateContext {
       client: { short_name: string };
       project: { start_date: string; end_date: string };
       report_date: string;
       team: Array<{ name: string; email: string }>;
       findings: Array<{
         title: string;
         severity: string;
         severity_color: string;
         finding_type: string;
         cvss_score: number;
         cvss_vector: string;
         affected_entities: string;
         description: string;
         impact: string;
         recommendation: string;
         replication_steps: string;
         references: string;
       }>;
       scope: Array<{ scope: string }>;
       totals: { findings: number };
     }
     ```
   - Mapping logic:
     - `client.short_name` = `report.project.client.shortName || report.project.client.name` (fallback to name if shortName empty)
     - `project.start_date` = formatted `report.project.startDate`
     - `project.end_date` = formatted `report.project.endDate`
     - `report_date` = formatted `report.creation`
     - `team` = `report.project.assignments.map(a => ({ name: a.user.name || a.user.username, email: a.user.email }))`
     - `findings` = mapped from `report.findings` with severity/type/CVSS flattened
     - `scope` = `report.project.scopes.map(s => ({ scope: s.scope }))`
     - `totals.findings` = `report.findings.length`
   - Note: Rich text fields (description, impact, etc.) contain HTML from GW. The actual HTML-to-DOCX rich text conversion happens in the Python Jinja2 renderer (Plan 04-04), not here. This mapper just passes the HTML strings through.
2. Export both `mapReportToTemplateContext` and `TemplateContext` type

### Task 3: Ghostwriter API Routes
**Commit:** `feat(ghostwriter): add API routes for GW report data and health`
**Files:**
- backend/src/routes/ghostwriter.ts -- new file
- backend/src/index.ts -- mount new router

**Steps:**
1. Create `backend/src/routes/ghostwriter.ts` with Router:
   - `GET /api/ghostwriter/report/:id`:
     - Requires auth
     - Validates `id` as positive integer (Zod)
     - Calls `fetchReportData(id)` then `mapReportToTemplateContext(report)`
     - Returns both raw GW data and mapped template context: `{ report: GWReport, templateContext: TemplateContext }`
     - Logs audit event for GW data fetch
   - `GET /api/ghostwriter/health`:
     - Requires auth
     - Calls `checkGhostwriterHealth()`
     - Returns `{ available: boolean, username?: string }`
   - Error responses: 404 if report not found, 503 if GW unavailable, 401 if GW auth invalid
2. Mount in `backend/src/index.ts`: `app.use('/api/ghostwriter', ghostwriterRouter)`
3. Handle case where GW env vars are not configured: return 503 with message "Ghostwriter not configured"
