## Context
Build the two reusable frontend components needed for document processing: a file upload component with drag-and-drop and validation, and a PDF preview component with page navigation. These components will be used by the Template Adapter (Phase 5) and Executive Report Generator (Phase 8). The backend APIs for document upload and PDF conversion are built in Plans 04-02 and 04-04. This plan focuses purely on the frontend: feature module, API hooks, and the two UI components.

## Tasks

### Task 1: Documents Feature Module (API + Types + Hooks)
**Commit:** `feat(documents): add frontend feature module with API client and hooks`
**Files:**
- frontend/src/features/documents/types.ts -- new file
- frontend/src/features/documents/api.ts -- new file
- frontend/src/features/documents/hooks.ts -- new file

**Steps:**
1. Create `frontend/src/features/documents/types.ts`:
   ```typescript
   export interface UploadResponse {
     id: string;
     filename: string;
     originalName: string;
     size: number;
     mimeType: string;
   }
   export interface PdfConversionJob {
     jobId: string;
     status: 'queued' | 'active' | 'completed' | 'failed';
     progress?: number;
     pdfUrl?: string;
     error?: string;
   }
   export interface DocumentPreviewResult {
     docxUrl: string;
     pdfJobId: string;
   }
   export interface GhostwriterReport {
     // Subset of GW data needed by frontend
     report: { id: number; title: string; creation: string };
     templateContext: Record<string, unknown>;
   }
   export interface GhostwriterHealth {
     available: boolean;
     username?: string;
   }
   ```
2. Create `frontend/src/features/documents/api.ts`:
   - `uploadDocument(file: File): Promise<UploadResponse>` -- Uses `apiUpload()` from `@/lib/api` to POST to `/api/documents/convert-pdf`
   - `getPdfJobStatus(jobId: string): Promise<PdfConversionJob>` -- GET `/api/documents/convert-pdf/${jobId}`
   - `downloadDocument(filename: string): string` -- Returns URL string for `/api/documents/download/${filename}`
   - `requestPreview(templatePath: string, reportId: number): Promise<DocumentPreviewResult>` -- POST `/api/documents/preview`
   - `getGhostwriterReport(reportId: number): Promise<GhostwriterReport>` -- GET `/api/ghostwriter/report/${reportId}`
   - `getGhostwriterHealth(): Promise<GhostwriterHealth>` -- GET `/api/ghostwriter/health`
3. Create `frontend/src/features/documents/hooks.ts`:
   - `useUploadDocument()` -- TanStack `useMutation` wrapping `uploadDocument`, includes `onProgress` callback
   - `usePdfJobStatus(jobId: string | null)` -- TanStack `useQuery` with polling (refetchInterval: 2000ms while status is 'queued' or 'active', stops when 'completed' or 'failed')
   - `useGhostwriterReport(reportId: number)` -- TanStack `useQuery` wrapping `getGhostwriterReport`
   - `useGhostwriterHealth()` -- TanStack `useQuery` wrapping `getGhostwriterHealth`
   - `useDocumentPreview()` -- TanStack `useMutation` wrapping `requestPreview`

### Task 2: FileUpload Reusable Component
**Commit:** `feat(ui): add reusable FileUpload component with drag-and-drop`
**Files:**
- frontend/src/components/ui/file-upload.tsx -- new file

**Steps:**
1. Create `frontend/src/components/ui/file-upload.tsx` as a reusable, headless-ish component:
   - Props interface:
     ```typescript
     interface FileUploadProps {
       onFileSelect: (file: File) => void;
       accept?: string; // MIME types, default ".docx,.pdf"
       maxSizeMB?: number; // default 50
       disabled?: boolean;
       isUploading?: boolean;
       progress?: number; // 0-100
       error?: string;
       className?: string;
     }
     ```
   - Drag-and-drop zone:
     - Uses native HTML5 drag events (dragenter, dragover, dragleave, drop)
     - Visual feedback: border highlight on drag-over, drop zone text changes
     - Also supports click-to-browse (hidden file input triggered by zone click)
   - Client-side validation (before calling onFileSelect):
     - File extension check against accept prop
     - File size check against maxSizeMB
     - MIME type check (basic: `application/vnd.openxmlformats...` for .docx, `application/pdf` for .pdf)
     - Show validation error inline (red text below drop zone)
   - Upload progress bar:
     - Show when `isUploading` is true
     - Animate progress from `progress` prop (0-100)
     - Use Tailwind for styling (consistent with existing UI)
   - States: idle, drag-over, uploading (with progress), error, disabled
   - Icons from lucide-react: Upload, File, AlertCircle
   - Styling: Uses existing Tailwind theme tokens (consistent with shadcn components)
2. Component is stateless for upload logic (controlled by parent via props) but manages its own drag state internally

### Task 3: PdfPreview Component with react-pdf
**Commit:** `feat(ui): add PdfPreview component with page navigation`
**Files:**
- frontend/src/components/ui/pdf-preview.tsx -- new file
- frontend/package.json -- add react-pdf dependency

**Steps:**
1. Install react-pdf: `cd frontend && npm install react-pdf`
2. Configure pdf.js worker: react-pdf requires a pdf.js worker. Set up via the standard approach:
   ```typescript
   import { pdfjs } from 'react-pdf';
   pdfjs.GlobalWorkerOptions.workerSrc = new URL(
     'pdfjs-dist/build/pdf.worker.min.mjs',
     import.meta.url,
   ).toString();
   ```
3. Create `frontend/src/components/ui/pdf-preview.tsx`:
   - Props interface:
     ```typescript
     interface PdfPreviewProps {
       url: string | null;
       isLoading?: boolean;
       error?: string;
       className?: string;
       onPageChange?: (page: number, totalPages: number) => void;
     }
     ```
   - Document loading: Use react-pdf's `<Document>` and `<Page>` components
   - Page navigation:
     - Previous/Next buttons (ChevronLeft, ChevronRight from lucide-react)
     - Page number display: "Page X of Y"
     - Direct page jump input (number field)
     - Keyboard navigation: Left/Right arrow keys for prev/next
   - Zoom controls:
     - Zoom in/out buttons (ZoomIn, ZoomOut from lucide-react)
     - Preset zoom levels: Fit width, 75%, 100%, 125%, 150%
     - Current zoom display as percentage
   - Loading state:
     - Skeleton loader while PDF loads (use existing Skeleton component from shadcn)
     - Page loading indicator for individual page renders
   - Error state:
     - Display error message with AlertCircle icon
     - Retry button
   - Layout: Toolbar at top (navigation + zoom), scrollable PDF area below
   - Responsive: PDF scales to container width in "fit width" mode
4. Add `react-pdf` CSS import for annotation layer (if needed) or disable annotation/text layers for performance

### Task 5: Component Accessibility and Polish
**Commit:** `feat(ui): add accessibility and keyboard support to document components`
**Files:**
- frontend/src/components/ui/file-upload.tsx -- update with a11y
- frontend/src/components/ui/pdf-preview.tsx -- update with a11y

**Steps:**
1. FileUpload accessibility:
   - Drop zone has `role="button"` and `tabIndex={0}`
   - Keyboard: Enter/Space triggers file browser
   - `aria-label` on the drop zone describing its purpose
   - Error messages linked via `aria-describedby`
   - Upload progress announced via `aria-live="polite"` region
   - Focus ring visible on keyboard focus (Tailwind `focus-visible:ring-2`)
2. PdfPreview accessibility:
   - Navigation buttons have `aria-label` ("Previous page", "Next page")
   - Page counter is an `aria-live="polite"` region
   - Keyboard navigation: arrow keys for page nav, +/- for zoom
   - Zoom buttons have `aria-label`
   - Page jump input has associated label
   - Focus management: after page change, focus stays on navigation controls
3. Both components:
   - Respect `prefers-reduced-motion` for animations (progress bar, page transitions)
   - High contrast: ensure all states are distinguishable in the dark theme
   - Test with keyboard-only navigation (no mouse)
