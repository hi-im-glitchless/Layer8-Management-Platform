## Objective

Rewrite the backend upload and sanitization pipeline to be HTML-centric. Upload converts DOCX to HTML via mammoth, sanitizes HTML text nodes with Presidio using incrementing placeholders, injects entity `<span>` tags, and stores the result in an overhauled session model. This replaces the paragraph-by-paragraph DOCX sanitization with HTML-native processing.

## Context

- `@backend/src/services/reportService.ts` -- current upload/sanitize pipeline (paragraph-based) to rewrite
- `@backend/src/services/reportWizardState.ts` -- current session model to overhaul (add HTML fields, remove DOCX fields)
- `@backend/src/routes/executiveReport.ts` -- routes to modify (merge sanitize into upload pipeline, update response shapes)
- `@backend/src/services/sanitization.ts` -- existing `sanitizeText()` and `desanitizeText()` functions to reuse for text node processing
- `@06.1-CONTEXT.md` decisions: #15 upload auto-triggers full pipeline, #18 session-scoped counter map, #22 replace in-place, #23 deny list merged into mapping table, #26 add HTML fields remove DOCX fields, #27 incrementing placeholders, #29 sanitize HTML text nodes
- Plan 06.1-A provides: `docxToHtml.ts` (mammoth converter), supplementary text extraction endpoint

### Task 2: Implement HTML text node sanitization with entity spans

**Files:**
- `backend/src/services/htmlSanitizer.ts` (new)

**What:** Create a new module that sanitizes HTML by walking text nodes, calling Presidio for each, and wrapping detected entities in `<span>` tags with data attributes. Uses a session-scoped counter map for incrementing placeholders.

Exports:
- `sanitizeHtmlTextNodes(html: string, sessionId: string, counterMap: Record<string, Record<string, number>>): Promise<SanitizeHtmlResult>`

`SanitizeHtmlResult` type:
```typescript
interface SanitizeHtmlResult {
  sanitizedHtml: string;
  entityMappings: EntityMapping[];
  updatedCounterMap: Record<string, Record<string, number>>;
  sanitizedParagraphs: SanitizedParagraph[]; // backward compat
  forwardMappings: Record<string, string>;
  reverseMappings: Record<string, string>;
}
```

Implementation:
1. Parse HTML string using a lightweight parser (regex-based or `node-html-parser`) to extract text nodes while preserving tags
2. For each text node: call existing `sanitizeText()` from `sanitization.ts`
3. For each detected entity: look up the counter map. If `originalValue` already seen for this `entityType`, reuse same index. Otherwise, increment counter and assign next index.
4. Generate placeholder: `[ENTITY_TYPE_N]` format (e.g., `[PERSON_1]`, `[IP_ADDRESS_3]`)
5. Replace entity text with: `<span class="entity entity-{type-lowercase}" data-entity-type="{TYPE}" data-placeholder="{placeholder}" data-original="{originalValue}">{placeholder}</span>`
6. Build EntityMapping array and updated counter map
7. Also build sanitizedParagraphs array (extract text-only paragraphs from HTML for Pass 1 extraction compatibility)

**Acceptance:**
- [ ] HTML input with PII produces HTML with entity `<span>` tags
- [ ] Same original value always gets same placeholder within a session
- [ ] Counter increments correctly for new unique values per entity type
- [ ] `data-entity-type`, `data-placeholder`, `data-original` attributes present on each span
- [ ] HTML structure (tags, attributes) is preserved -- only text content changes
- [ ] Returns both entity mappings and backward-compatible paragraph arrays

**Commit:** `feat(report): add HTML text node sanitization with incrementing entity spans`

### Task 3: Rewrite upload pipeline to DOCX->HTML->sanitize->extract flow

**Files:**
- `backend/src/services/reportService.ts` (modify)

**What:** Rewrite the `uploadReport()` function to implement the new pipeline: mammoth DOCX->HTML -> HTML sanitization -> supplementary text extraction. Also modify `sanitizeReport()` to work with HTML instead of paragraphs (used when entity mappings change).

Updated `uploadReport()` flow:
1. Save DOCX to disk (keep for supplementary extraction)
2. Convert DOCX to HTML via `convertDocxToHtml()` from `docxToHtml.ts`
3. Detect language from HTML text (same as before, use first ~500 chars)
4. Sanitize HTML text nodes via `sanitizeHtmlTextNodes()`
5. Extract supplementary text (headers/footers) via Python endpoint
6. Sanitize supplementary text segments alongside body
7. Store results: `uploadedHtml`, `sanitizedHtml`, `entityMappings`, `entityCounterMap`, `supplementaryText`, `sanitizedParagraphs`, `sanitizationMappings`
8. Return `{ sessionId, detectedLanguage }`

Remove `sanitizeReport()` as a separate operation -- sanitization is now part of the upload pipeline (decision #15). Keep the function but have it re-run HTML sanitization (used when user modifies entity mappings from the frontend).

Remove `updateDenyList()` entirely -- deny list is replaced by entity mapping edits (decision #23).

**Acceptance:**
- [ ] `uploadReport()` calls mammoth, sanitizes HTML, extracts supplementary text
- [ ] Session state has `uploadedHtml`, `sanitizedHtml`, `entityMappings` populated after upload
- [ ] `sanitizeReport()` re-runs HTML sanitization with current counter map
- [ ] `updateDenyList()` removed
- [ ] Language detection works from HTML text content
- [ ] Supplementary text stored in session

**Commit:** `feat(report): rewrite upload pipeline for DOCX-to-HTML-to-sanitize flow`
