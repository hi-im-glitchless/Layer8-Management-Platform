## Tasks

### Task 1: Rewrite StepReview with HTML preview and de-sanitize toggle

**Files:**
- `frontend/src/features/executive-report/components/StepReview.tsx` (rewrite)

**What:** Replace the react-pdf PdfPreview-based review with HtmlReportPreview showing the LLM-generated HTML. Add a de-sanitization toggle button that performs frontend-only find-replace of placeholders with original values from entity mappings.

Layout (same split as template adapter):
- Left/top 60%: HTML preview in iframe (HtmlReportPreview component)
- Right/bottom 40%: ReportChatPanel (existing, keep)
- Toolbar above preview: De-sanitize toggle button, Regenerate button

De-sanitize toggle:
- Button: "Show Real Values" / "Show Sanitized" with toggle state
- When toggled ON: take the `generatedHtml` from session, find all `[ENTITY_TYPE_N]` placeholders, replace with original values from `entityMappings` array. Render the de-sanitized HTML in the iframe.
- When toggled OFF: render the original `generatedHtml` (with placeholders)
- Implementation: pure frontend string replacement. Build a `Record<placeholder, originalValue>` from entity mappings, then `html.replaceAll(placeholder, originalValue)` for each.
- Visual indicator: badge showing "De-sanitized" or "Sanitized" state
- Default: sanitized (matches what LLM generated)

Chat corrections:
- Same ReportChatPanel with same useReportChat hook
- `onSectionUpdate` callback: when a section_update SSE event arrives, refetch session to get updated `generatedHtml`, update preview
- If de-sanitize toggle is ON when correction arrives, re-apply de-sanitization to the new HTML

Remove: PdfPreview import, PDF polling logic (useReportPreviewStatus for intermediate preview), PDF conversion status display. The HTML preview is immediate -- no conversion needed for preview. PDF conversion only happens at download time.

**Acceptance:**
- [ ] HTML preview renders `generatedHtml` from session in iframe
- [ ] De-sanitize toggle replaces placeholders with real values (frontend-only)
- [ ] Toggle state persists during the session (local state)
- [ ] Chat corrections update the HTML preview in real-time
- [ ] De-sanitization re-applied after correction if toggle is ON
- [ ] No PdfPreview or PDF polling in the Review step
- [ ] "Satisfied -- Continue" button advances to Download step

**Commit:** `feat(report): rewrite StepReview with HTML preview and de-sanitize toggle`

### Task 2: Update StepDownload for PDF-only output

**Files:**
- `frontend/src/features/executive-report/components/StepDownload.tsx` (modify)

**What:** Simplify the download step to offer PDF download only (no DOCX). The PDF is generated from the de-sanitized HTML via Gotenberg at download time.

Flow:
1. User clicks "Download PDF" button
2. Frontend takes `generatedHtml` from session, applies de-sanitization (same logic as toggle), sends de-sanitized HTML to backend
3. Backend sends HTML to Gotenberg `/forms/chromium/convert/html` with `waitDelay` parameter (e.g., 3000ms for Chart.js rendering)
4. Gotenberg returns PDF bytes
5. Backend streams PDF to frontend as download

UI changes:
- Remove DOCX download button and link
- Single "Download PDF" button with loading state
- Show PDF file size after generation
- "Start New Report" button (keep)
- Summary card showing report metadata (keep)

New backend endpoint: `POST /api/report/download-pdf` accepts `{ sessionId }`, performs de-sanitization server-side (using session entity mappings), sends to Gotenberg, returns PDF bytes as response.

Alternative: use existing PDF queue but trigger from the download step with de-sanitized HTML. The key change: de-sanitization happens at PDF generation time, not at HTML generation time.

**Acceptance:**

- [ ] Loading state while PDF generates

### Task 3: Add Gotenberg HTML-to-PDF conversion function

**Files:**
- `backend/src/services/pdfQueue.ts` (modify)
- `backend/src/routes/executiveReport.ts` (modify)

**What:** Add a new function for converting HTML to PDF via Gotenberg's Chromium-based HTML input endpoint. This is different from the existing DOCX-to-PDF LibreOffice conversion.

New function in `pdfQueue.ts`:
```typescript
export async function convertHtmlToPdf(html: string, waitDelay?: number): Promise<Buffer>
```

Implementation:
- POST to `${GOTENBERG_URL}/forms/chromium/convert/html`
- Send as multipart form with the HTML as a file named `index.html`
- Set `waitDelay` to allow Chart.js to render (default: 3000ms)
- Set page format to A4 with margins matching CSS @page rules
- Return PDF as Buffer

New route `POST /api/report/download-pdf`:
1. Get session, verify ownership
2. Apply de-sanitization to `generatedHtml` using entity mappings
3. Call `convertHtmlToPdf()` with de-sanitized HTML
4. Stream PDF buffer as response with `Content-Type: application/pdf`
5. Set `Content-Disposition: attachment` with report filename

Also update the generate pipeline: after HTML assembly, convert to PDF using this function (for the preview URL). Store the PDF URL in `reportPdfUrl` for the Review step's "Satisfied" button validation.

**Acceptance:**
- [ ] `convertHtmlToPdf()` function sends HTML to Gotenberg Chromium endpoint
- [ ] waitDelay parameter included for Chart.js rendering
- [ ] `POST /api/report/download-pdf` endpoint works
- [ ] De-sanitization applied before PDF generation
- [ ] PDF response has correct Content-Type and Content-Disposition headers
- [ ] A4 page format with correct margins

**Commit:** `feat(report): add Gotenberg HTML-to-PDF conversion for report download`
