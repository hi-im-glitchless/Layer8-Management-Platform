## Objective

Rewrite the Sanitize & Review step (Step 2) of the executive report wizard. Replace the paragraph-diff view and deny list editor with an interactive HTML preview in a sandboxed iframe, a toggleable entity mapping table, and a floating popover for text-selection-based entity creation. This is the primary UX improvement of Phase 6.1.

## Tasks

### Task 1: Create HtmlReportPreview iframe component

**Files:**
- `frontend/src/features/executive-report/components/HtmlReportPreview.tsx` (new)

**What:** Create a reusable iframe-based HTML preview component used in both the Sanitize and Review steps. The iframe uses `srcdoc` for CSS isolation from the app UI and `sandbox="allow-scripts"` to allow Chart.js execution.

Props:
```typescript
interface HtmlReportPreviewProps {
  html: string;
  className?: string;
  onTextSelection?: (selection: { text: string; range: { start: number; end: number } }) => void;
}
```

Implementation:
- Render `<iframe srcdoc={html} sandbox="allow-scripts" />` with full-height styling
- Inject a postMessage script into the HTML that listens for `mouseup` events in the iframe and sends selection data to the parent window
- Parent component listens for `message` events, validates origin, and calls `onTextSelection` callback
- Style: `w-full border rounded-lg` with min-height 600px, auto-resize based on content height
- The iframe should inject the report-template.css styles inline into the HTML `<head>` (or the HTML already has them from the skeleton)

**Acceptance:**
- [ ] Iframe renders HTML content with CSS isolation
- [ ] Text selection in iframe triggers `onTextSelection` callback with selected text
- [ ] `sandbox="allow-scripts"` attribute present for Chart.js
- [ ] Component handles empty/null HTML gracefully (show placeholder)
- [ ] Auto-resizes to content height

**Commit:** `feat(report): add HtmlReportPreview iframe component with text selection`

### Task 2: Create EntityMappingTable component

**Files:**
- `frontend/src/features/executive-report/components/EntityMappingTable.tsx` (new)

**What:** Create the 4-column mapping table that replaces both the SanitizationDiffView and DenyListEditor. Shows all detected (and manually added) entity mappings.

Props:
```typescript
interface EntityMappingTableProps {
  mappings: EntityMapping[];
  onEditType: (index: number, newType: string) => void;
  onDelete: (index: number) => void;
  isUpdating: boolean;
}
```

Columns:
1. **Original Value** -- the real text that was sanitized (truncated to 40 chars with tooltip for full value)
2. **Placeholder** -- the placeholder string like `[PERSON_1]` (read-only, auto-generated)
3. **Entity Type** -- dropdown select with ~15 entity types: PERSON, ORGANIZATION, LOCATION, EMAIL_ADDRESS, PHONE_NUMBER, IP_ADDRESS, URL, DATE_TIME, CREDIT_CARD, HOSTNAME, DOMAIN, MAC_ADDRESS, CVE_ID, NETWORK_RANGE, CUSTOM
4. **Actions** -- Edit (pencil icon, triggers type dropdown) and Delete (X icon, removes mapping and restores original text)

Entity type constants array exported from a shared `entityTypes.ts` file.

Behavior:
- Changing entity type via dropdown calls `onEditType` which triggers backend re-sanitization
- Deleting a mapping calls `onDelete` which removes the entity span from HTML
- Manual entries (isManual=true) shown with a subtle badge
- Table is scrollable with sticky header when mappings exceed viewport
- Empty state: "No entities detected. Select text in the preview to add mappings."

**Acceptance:**
- [ ] 4 columns render correctly with entity data
- [ ] Entity type dropdown contains all ~15 types
- [ ] Edit and delete callbacks fire correctly
- [ ] Scrollable with sticky header
- [ ] Shows manual badge for user-added mappings
- [ ] Empty state message when no mappings

**Commit:** `feat(report): add EntityMappingTable with edit/delete entity actions`

**Acceptance:**
