## Objective

Extend the OKLch theme system with semantic color tokens (success, warning, info) and migrate all hardcoded Tailwind color classes in core layout and admin components to use theme variables. This is the foundation that enables consistent theming across the entire app.

## Context

- `@frontend/src/index.css` -- theme system with 40+ CSS custom properties in OKLch format, missing semantic tokens for success/warning/info
- `@frontend/src/routes/Profile.tsx` -- hardcoded oklch avatar bg (line 199), bg-green-500 TOTP dot (line 315), bg-yellow-500 warning (line 366)
- `@frontend/src/components/layout/Header.tsx` -- bg-orange-500 admin badge (line 63)
- `@frontend/src/components/layout/Sidebar.tsx` -- border-blue-500 active indicator (line 134), text-blue-500 collapsed icon (line 109)
- `@frontend/src/components/admin/UserManagement.tsx` -- bg-green-600 Active badge (line 127), bg-blue-600 MFA badge (line 140)
- `@frontend/src/components/admin/LLMSettings.tsx` -- bg-green-500/bg-red-500 status dots (lines 168, 217), text-red-600 stop button (line 198), orange warning box (line 234)
- `@frontend/src/components/admin/SessionManagement.tsx` -- text-green-600 activity icon (line 120)
- `@frontend/src/components/auth/PasswordChange.tsx` -- bg-red-500/bg-yellow-500/bg-green-500 strength meter (lines 67-69)

## Tasks

### Task 1: Add semantic color tokens to theme system

**Scope:** `frontend/src/index.css`

**Action:** Add success, warning, and info semantic color tokens to the theme:

1. In the `@theme inline` block, add:
   - `--color-success: var(--success)`, `--color-success-foreground: var(--success-foreground)`
   - `--color-warning: var(--warning)`, `--color-warning-foreground: var(--warning-foreground)`
   - `--color-info: var(--info)`, `--color-info-foreground: var(--info-foreground)`

2. In `:root` (light mode), add:
   - `--success: oklch(0.55 0.16 145)` (green), `--success-foreground: oklch(1 0 0)` (white)
   - `--warning: oklch(0.75 0.15 85)` (amber/yellow), `--warning-foreground: oklch(0.25 0.05 85)` (dark amber)
   - `--info: oklch(0.55 0.16 250)` (blue, matches primary), `--info-foreground: oklch(1 0 0)` (white)

3. In `.dark`, add:
   - `--success: oklch(0.65 0.16 145)` (brighter green for dark bg), `--success-foreground: oklch(0.98 0 0)`
   - `--warning: oklch(0.75 0.15 85)` (amber), `--warning-foreground: oklch(0.98 0 0)`
   - `--info: oklch(0.65 0.16 250)` (brighter blue, matches dark accent), `--info-foreground: oklch(0.98 0 0)`

**Acceptance:** Tailwind classes `bg-success`, `text-success`, `bg-warning`, `text-warning`, `bg-info`, `text-info` (and foreground variants) work in both light and dark mode.
