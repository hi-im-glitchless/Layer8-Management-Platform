## Objective

Transform the Dashboard from skeleton placeholder cards into a functional activity hub showing real session data, a personalised greeting, and quick-action cards. Uses existing TanStack Query hooks — no new backend endpoints needed.

## Context

- `@frontend/src/routes/Dashboard.tsx` -- current stub with 3 Skeleton cards, "Welcome to Layer8" text (will be rebranded by Plan 07-02)
- `@frontend/src/features/adapter/hooks.ts` -- `useActiveSession()` returns active adapter wizard session (sessionId, currentStep, templateFile.originalName, updatedAt)
- `@frontend/src/features/executive-report/hooks.ts` -- `useActiveReportSession()` returns active report session (sessionId, currentStep, uploadedFile.originalName, updatedAt)
- `@frontend/src/features/auth/hooks.ts` -- `useAuth()` returns `{ user }` with `displayName`, `username`, `isAdmin`
- Existing session endpoints: `GET /api/adapter/session` (returns single active session or null), `GET /api/report/session` (returns single active session or null)
- Dashboard card design pattern: Card + CardHeader + CardContent from shadcn/ui (already imported)
- Navigation: `useNavigate` from react-router-dom for quick-action card clicks

## Tasks

### Task 1: Add personalised welcome section

**Scope:** `frontend/src/routes/Dashboard.tsx`

**Action:**
- Import `useAuth` from `@/features/auth/hooks`
- Replace the static heading with a personalised greeting:
  - If user has displayName: "Welcome back, {displayName}"
  - Fallback: "Welcome back, {username}"
- Keep subtitle as "Your AI-powered security reporting hub" (theme-consistent `text-muted-foreground`)
- Use time-of-day greeting: "Good morning/afternoon/evening, {name}"

**Acceptance:** Dashboard shows personalised greeting using the logged-in user's display name. Falls back to username if no display name set.

### Task 2: Build quick-action cards

**Scope:** `frontend/src/routes/Dashboard.tsx`

**Action:**
- Import `useNavigate`, `FileCode`, `FileText` from lucide-react
- Create two quick-action cards in a 2-column grid:
  1. "New Template Adaptation" — FileCode icon, description "Upload and adapt a DOCX template with AI-powered placeholder mapping", onClick navigates to `/template-adapter`
  2. "New Executive Report" — FileText icon, description "Generate a professional executive report from your technical findings", onClick navigates to `/executive-report`
- Cards use theme classes: `bg-card`, `hover:bg-accent/10`, `border`, `cursor-pointer`, `transition-colors`

**Acceptance:** Two clickable quick-action cards render on the dashboard. Clicking each navigates to the correct route.

### Task 3: Wire active session display cards

**Scope:** `frontend/src/routes/Dashboard.tsx`

**Action:**
- Import `useActiveSession` from `@/features/adapter/hooks` and `useActiveReportSession` from `@/features/executive-report/hooks`
- Import `formatDistanceToNow` from `date-fns` for relative timestamps
- Create a "Recent Activity" section below the quick-action cards
- For each active session (adapter and/or report), show a card with:
  - File name (`templateFile.originalName` or `uploadedFile.originalName`)
  - Current step label (map step number to human-readable label)
  - Relative timestamp ("2 hours ago")
  - "Continue" button linking to the respective wizard page
- If both sessions are null, show empty state

**Acceptance:** When an adapter session exists, a card shows the template filename, current step, and relative timestamp. Same for report sessions. Cards link to the correct wizard pages.

### Task 4: Add loading and empty states

**Scope:** `frontend/src/routes/Dashboard.tsx`

**Action:**
- While session queries are loading, show Skeleton components (reuse existing Skeleton import)
- When no active sessions exist, show an empty state card:
  - Illustration or icon (e.g., `Inbox` from lucide-react)
  - "No active sessions" heading
  - "Start a new template adaptation or executive report to see your activity here." description
  - Theme-consistent muted colors
- Handle error states gracefully (show cards as empty rather than crashing)

**Acceptance:** Dashboard shows skeletons during loading, empty state when no sessions, and gracefully handles API errors without crashing.
