# Phase 7 Research: UI Polish

**Date:** 2026-02-16
**Scope:** React/TypeScript frontend polish, OKLch theme migration, rebranding, dead code cleanup
**Status:** Ready to plan

## Findings

### 1. Dashboard Implementation Status
- **Current state:** Dashboard skeleton with 3 placeholder cards (Recent Reports, Template Adapters, Activity) containing Skeleton loaders
- **Pattern:** Follows React dashboard best practices—card-based layout (2-3 columns responsive), CardComponent + CardHeader/Content/Description structure
- **Gap:** No actual data binding; all cards are UI stubs. Real data requires wiring activity feeds, session history, and quick-action patterns
- **Modern pattern (2025):** Activity dashboards blend personalized greetings, statistics cards, and recent activity streams with modular, reusable card components (Recharts 3.0 adds accessibility + auto-sizing)

### 2. OKLch Theme System Status
- **Current state:** Theme is **well-architected**—Tailwind v4 using `@theme inline` directive with 40+ CSS custom properties (--color-*, --radius-*)
- **Theme values:** All colors defined in OKLch format in `:root` and `.dark` (light/dark variants correctly implemented)
- **Problem:** 20+ hardcoded Tailwind color utilities in component files (bg-blue-500, border-orange-200, bg-green-900/30, text-yellow-800, etc.)
- **Scope of hardcodes:** Found in Header, LLMSettings, UserManagement, OnboardingWizard, PasswordChange, Adapter components, InteractivePdfViewer, MappingOverlayCard
- **CSS colors in generated HTML:** HtmlReportPreview contains 15+ hardcoded hex values (#1a1a1a, #e5e7eb, #d1d5db, #3b82f6) for report styling
- **Audit approach:** Search `frontend/src` for `bg-`, `text-`, `border-` + named Tailwind colors (blue, green, red, etc.), plus inline style `color:` and `background:` in JSX

### 3. Rebranding Status
- **Text instances to replace:** "Layer8" appears in:
  - Dashboard.tsx: "Welcome to Layer8. Your security reporting hub."
  - OnboardingWizard.tsx: "Welcome to Layer8" + "You can now use Layer8 to streamline..."
  - Sidebar.tsx: `alt="Layer8"` (logo alt text)
  - Login.tsx: Two `alt="Layer8"` attributes (logo)
  - auth.ts (backend): QR code issuer label (not visible but affects MFA experience)
  - auth.test.ts: Test assertion comment
- **Target:** "AI Template Regenerator"
- **Scope:** Frontend text labels + alt attributes; backend issuer can stay as-is or change (UX decision)
- **Pattern:** Centralized text management (React Context + JSON config) recommended for scale, but 6-8 hardcoded instances can be done via find/replace safely with TypeScript type checking

### 4. Dead Code Detection State
- **Tool recommendation:** Knip (2025 standard) > ts-prune. Knip handles:
  - Unused exports (functions, components, types)
  - Unused npm dependencies
  - Unused CSS classes (via CSS analyzer plugin)
  - Monorepo support
- **Setup:** `npm install -D knip` + `knip --config` to generate config, then `knip --reporter json` for automated analysis
- **Expected finds:** Likely unused imports in test files, old feature toggles, dead CSS classes from removed features

## Relevant Patterns

### React Dashboard with Real Data
1. **Session history table:** Show recent report generation + template adaptation runs with timestamp, status, file name
2. **Quick-action cards:** Next Steps (e.g., "Start new adaptation", "Download last report") with click handlers
3. **Activity stream:** Real-time or polling-based feed of system events (file uploads, report completions)
4. **Stats row:** Total reports generated, templates processed, average runtime
5. **Empty state:** Fallback UI when no history exists (common for new users)

**Implementation:** Wire dashboard to TanStack Query hooks (already established pattern in codebase), use `useQuery` to fetch `/api/sessions` and `/api/reports` endpoints.

### Color Migration Strategy
1. **Audit phase:** Grep for all non-theme colors
2. **Map to theme:** Create mapping table (bg-blue-500 → bg-primary, bg-green-100 → semantic color like `bg-success-light`)
3. **Extend theme if needed:** For semantic colors (success, warning, error) not in current palette, add CSS custom properties
4. **Update components:** Replace hardcoded classes with theme-based equivalents
5. **Validation:** Check contrast ratios in both light + dark modes (OKLch perceptual uniformity helps here)

**Current theme tokens ready for use:**
- `bg-primary`, `text-primary-foreground` (cool blue accent)
- `bg-destructive` (red for danger)
- `bg-accent`, `text-accent-foreground` (secondary emphasis)
- `bg-secondary`, `text-secondary-foreground` (subtle surface)
- Chart colors (--color-chart-1 through 5) for data viz

### Systematic Rebranding
1. **Search:** `grep -r "Layer8" frontend/src backend/src --include="*.tsx" --include="*.ts"`
2. **Categorize:** UI text vs. internal labels vs. comments
3. **Replace:** Use sed or IDE find/replace with proper scoping (don't touch node_modules, .git)
4. **Test:** Full UI walkthrough + search again to catch variants (lowercase, abbreviations)
5. **Comments:** Update CSS comments mentioning "Layer8" for consistency (lines 51, 86 in index.css)

### Dead Code Removal Workflow
1. **Run Knip:** Generate initial report
2. **Review findings:** Verify each unused export is truly dead (not used via dynamic imports or re-exports)
3. **Remove incrementally:** Delete in small commits grouped by feature area
4. **Test:** Run full test suite after each commit to catch breakage

## Risks

| Risk | Impact | Mitigation |
|------|--------|-----------|
| **Dashboard data binding incomplete** | User sees empty activity, poor UX | Ensure API endpoints (/sessions, /reports) exist; use fallback "no data" states |
| **Color migration introduces contrast issues** | Accessibility regression in dark mode | Test color pairs against WCAG AA (4.5:1 text, 3:1 non-text) with OKLch perceptual validation |
| **Branding replace misses instances** | Incomplete rebrand, confusion | Use case-sensitive regex; test all user-facing pages + alt text |
| **Dead code removal breaks subtle dependencies** | Runtime errors post-cleanup | Run full test suite + E2E after each removal batch; verify no re-exports used elsewhere |
| **HtmlReportPreview hardcoded colors** | Generated reports don't respect theme | Migrate to CSS var() or compute colors from CSS custom properties in React |
