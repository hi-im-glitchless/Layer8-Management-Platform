## Objective
Modify the Python prompt builder to accept locked/unknown section splits from the backend and produce inline annotations in document order. The LLM sees `[RESOLVED: finding.title]` for known mappings (positional context) and `[NEEDS MAPPING]` for sections it must analyze. Remove the old advisory KB context block (zone distribution, repetition hints, confidence notes) and the token budget enforcer. Keep only the blueprints section for structural pattern context. This directly implements the "inline annotations over separate KB block" decision.

  ```python
  def _build_blueprints_section(kb_context: KBContext) -> str | None:
      """Build standalone blueprints section for structural pattern context.

      Extracted from the old KB context block. Returns None if no blueprints.
      """
      if not kb_context or not kb_context.blueprints:
          return None

      lines = ["## Structural Blueprints\n"]
      for bp in kb_context.blueprints:
          marker_names = [f"{m.get('gwField', '?')}" for m in bp.markers]
          markers_str = ", ".join(marker_names)
          anchor = f" (anchor: {bp.anchor_style})" if bp.anchor_style else ""
          lines.append(
              f"  {bp.pattern_type.capitalize()}: [{markers_str}] "
              f"in zone {bp.zone}{anchor}"
          )
      return "\n".join(lines)
  ```
