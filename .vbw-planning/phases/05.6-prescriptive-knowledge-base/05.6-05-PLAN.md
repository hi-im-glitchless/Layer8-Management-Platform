## Objective
Add purely visual KB indicators to the mapping table: a "KB" badge on locked rows, muted row styling for KB-sourced entries, and a summary banner showing the locked/LLM split counts. Zero functional changes to the mapping table's data model, event handlers, or the PDF pipeline. The source field on MappingEntry (added in Plan 02) drives all styling decisions via CSS conditional classes. This implements the "KB badge on rows" and "summary banner" acceptance criteria.

### Task 4: Add Muted Row Styling for KB-Sourced Entries
**File(s):** `frontend/src/features/adapter/components/MappingTable.tsx`
**Action:** Modify
**Details:**
- In the TableRow className (lines 291-295), add a muted styling condition for KB-sourced entries:
  ```tsx
  <TableRow
    key={`entry-${entry._idx}`}
    tabIndex={0}
    className={cn(
      'cursor-pointer transition-all duration-300',
      isHighlighted && 'ring-2 ring-primary/60 bg-primary/5 animate-pulse',
      !isHighlighted && entry.source === 'kb' && 'bg-muted/20 opacity-75',
      !isHighlighted && entry.source !== 'kb' && 'hover:bg-muted/50',
    )}
    onClick={() => { ... }}
  >
  ```
- The muted styling uses `bg-muted/20 opacity-75`:
  - `bg-muted/20` gives a subtle background tint
  - `opacity-75` slightly dims the entire row
- The highlight state (isHighlighted) takes priority over muted styling (when user clicks a row, it shows the full highlight ring regardless of source)
- hover:bg-muted/50 only applies to non-KB rows (KB rows already have a background)
- All interactive behaviors (click, double-click, keyboard, edit, delete) remain COMPLETELY unchanged. Only CSS classes are modified.
**Commit:** `style(adapter): add muted row styling for KB-sourced mapping entries`
