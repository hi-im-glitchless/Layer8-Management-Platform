## What Was Built

Unified the auto-map and regeneration pipelines so both use the same LLM placement logic (regenerateWithLLM). autoMapTemplate() now calls regenerateWithLLM() for Pass 2 instead of applyInstructions(), meaning upload and user-edit flows share identical placement code paths. PlacementPromptResponse extended with zone_map (paragraph_index to zone) on both Python (Pydantic model + endpoint) and TypeScript (interface + wizard state storage). persistMappingsToKB() now derives zone from the cached zone_map instead of hardcoding 'unknown', so KB entries store actual zones (body, cover, header, footer, table_cell). regenerateWithLLM() gained a source parameter ('auto-map' | 'reapply' | 'correction') for pipeline observability logging.
