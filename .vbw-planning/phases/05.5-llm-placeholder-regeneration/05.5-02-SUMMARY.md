## What Was Built

LLM placement pipeline orchestration in the TypeScript backend. New `regenerateWithLLM()` function in templateAdapter.ts sends the DOCX + mapping plan through a 6-step pipeline: build-placement-prompt (Python) -> LLM call (8192 tokens, template-adapter model) -> validate-placement (Python) -> apply (Python) -> save DOCX -> update wizard state. Both the POST /reapply route and processCorrectionChat() now use this function instead of the old mechanical reapplyFromMappingPlan(). WizardState.adaptation extended with placementWarnings (string[]) to surface skip/mismatch warnings from validation. Frontend ReapplyResponse type updated to include placementWarnings.
