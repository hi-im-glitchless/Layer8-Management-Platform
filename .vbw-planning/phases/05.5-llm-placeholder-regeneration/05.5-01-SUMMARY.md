## What Was Built

Python-side foundation for LLM-powered placeholder placement. A zone-aware prompt builder produces structured DOCX representations grouped by zone (cover, body, header, footer, table_cell) with paragraph indices, styles, and heading levels. Mapped paragraphs get full text (200 chars) with context; unmapped get truncated (50 chars) for token efficiency. The placement prompt has 4 sections: zone map, mapping entries, InstructionSet output format with confidence, and placement rules per marker type. A validation endpoint checks LLM-generated placement instructions against actual DOCX paragraphs -- bounds checking, original_text substring verification with text-based relocation fallback, marker_type/action validation, and confidence threshold filtering (<0.5). Validated instructions are packaged as an InstructionSet compatible with the existing POST /apply endpoint.
