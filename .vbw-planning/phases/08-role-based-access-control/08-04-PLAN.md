## Objective

Migrate all backend route files from the binary `isAdmin` checks to the new `requireRole(minimumRole)` middleware. Apply the access matrix: ADMIN-only for user management/admin panel/deny list, MANAGER+ for template adapter and executive report, and update the auth route to store/return role instead of isAdmin. Add session invalidation when an admin changes a user's role.

## Context

- `@backend/src/routes/auth.ts` -- login sets `req.session.isAdmin = user.isAdmin` (line 99); GET /me selects and returns `isAdmin` (lines 406, 416); both need role
- `@backend/src/routes/users.ts` -- uses `requireAdmin` (line 11); Zod schemas accept `isAdmin: z.boolean()` (lines 54, 118); PUT handler has self-demotion guard for isAdmin (line 124); needs role + session invalidation
- `@backend/src/routes/admin.ts` -- uses `requireAdmin` via `router.use(requireAdmin)` (line 49); simple swap to requireRole
- `@backend/src/routes/denyList.ts` -- uses `requireAdmin` on management routes (lines 36, 52, 94, 130, 151); simple swap
- `@backend/src/routes/audit.ts` -- uses `requireAdmin` on export/verify/purge (lines 74, 108, 129); GET / uses `session.isAdmin` for filtering (line 16); needs role check
- `@backend/src/routes/templateAdapter.ts` -- uses `requireAuth` on all routes; needs upgrade to `requireRole('MANAGER')`
- `@backend/src/routes/executiveReport.ts` -- uses `requireAuth` on all routes; needs upgrade to `requireRole('MANAGER')`
- `@backend/src/middleware/auth.ts` -- Plan 02 already provides `requireRole` and deprecated `requireAdmin` alias
- `@backend/src/services/session.ts` -- Plan 02 already provides `invalidateUserSessions`

## Tasks

### Task 1: Update auth routes (login + /me)

**Scope:** `backend/src/routes/auth.ts`

**Action:**

1. **POST /login** (line 97-99): Replace session setup:
   ```typescript
   // Before:
   req.session.isAdmin = user.isAdmin;
   // After:
   req.session.role = user.role;
   ```

2. **GET /me** (line 401-413): Update the Prisma select and response:
   ```typescript
   select: {
     id: true,
     username: true,
     displayName: true,
     avatarUrl: true,
     role: true,          // was: isAdmin: true
     totpEnabled: true,
     createdAt: true,
     lastLoginAt: true,
   },
   ```
   The response already returns the full user object, so the API contract changes automatically: `{ role: 'ADMIN' }` instead of `{ isAdmin: true }`.

3. Remove any `isAdmin` imports or references in this file.

**Acceptance:** Login stores `role` in session. GET /me returns `role` field. No `isAdmin` anywhere in auth.ts.

### Task 2: Update user management routes

**Scope:** `backend/src/routes/users.ts`

**Action:**

1. **Import changes**: Replace `requireAdmin` with `requireRole` from middleware. Import `invalidateUserSessions` from session service:
   ```typescript
   import { requireRole } from '../middleware/auth.js';
   import { invalidateUserSessions } from '../services/session.js';
   ```

2. **Router-level middleware** (line 11): Replace `router.use(requireAdmin)` with `router.use(requireRole('ADMIN'))`.

3. **GET /** (line 19-33): Update Prisma select — replace `isAdmin: true` with `role: true`.

4. **POST /** create user (lines 47-55): Update Zod schema:
   ```typescript
   const createUserSchema = z.object({
     username: z.string().min(3).max(50).regex(/^[a-zA-Z0-9_]+$/),
     password: z.string().min(8),
     role: z.enum(['NORMAL', 'PM', 'MANAGER', 'ADMIN']).optional().default('NORMAL'),
   });
   ```
   Update `prisma.user.create` data to use `role: validated.role`. Update the select in create response to return `role: true` instead of `isAdmin: true`.

5. **PUT /:id** update user (lines 110-155):
   - Update Zod schema: replace `isAdmin: z.boolean().optional()` with `role: z.enum(['NORMAL', 'PM', 'MANAGER', 'ADMIN']).optional()`.
   - Self-demotion guard (line 124): Replace `validated.isAdmin === false` check with:
     ```typescript
     if (id === session.userId && validated.role && validated.role !== 'ADMIN') {
       return res.status(400).json({ error: 'Cannot demote your own admin role' });
     }
     ```
   - **Session invalidation on role change**: After the `prisma.user.update` call, check if role was changed. If so, call `invalidateUserSessions(id)` to force re-login:
     ```typescript
     if (validated.role) {
       await invalidateUserSessions(id);
     }
     ```
   - Update select in response to return `role: true`.

6. Update all `select` objects in this file to use `role: true` instead of `isAdmin: true`.

**Acceptance:** All user CRUD operations use `role` field. Role change triggers session invalidation. Self-demotion guard works for role. API returns `role` instead of `isAdmin`.

## Success Criteria

- Login stores `role` in session, /me returns `role` in API response
- User CRUD uses `role` field with Zod enum validation
- Role change triggers immediate session invalidation via Redis
- Self-demotion guard prevents admin from removing own ADMIN role
- Admin panel, user management, deny list: ADMIN only
- Template adapter, executive report: MANAGER+
- Audit log: everyone sees own, ADMIN sees all
- All other authenticated routes: any authenticated user
- Zero `isAdmin` references in backend routes
