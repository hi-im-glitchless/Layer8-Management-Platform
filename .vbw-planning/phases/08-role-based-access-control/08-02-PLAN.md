## Objective

Build the RBAC infrastructure layer: update session type definitions to use `role` instead of `isAdmin`, create the `requireRole(minimumRole)` middleware factory with hierarchical enforcement, and add a session invalidation function to force re-login when roles change.

## Context

- `@backend/src/types/express.d.ts` -- SessionData interface has `isAdmin: boolean` at line 7; needs `role: string`
- `@backend/src/middleware/auth.ts` -- exports `requireAuth`, `requireAdmin`, `requirePendingTOTP`; requireAdmin checks `req.session.isAdmin` at line 37; needs replacement with `requireRole` factory
- `@backend/src/services/session.ts` -- manages Redis sessions (getActiveSessions, terminateSession); needs `invalidateUserSessions(userId)` for forced re-login on role change

## Tasks

### Task 1: Update session type definitions

**Scope:** `backend/src/types/express.d.ts`

**Action:**

Replace `isAdmin: boolean` with `role: string` in the SessionData interface:

```typescript
interface SessionData {
  userId: string;
  username: string;
  role: string;          // was: isAdmin: boolean
  totpVerified: boolean;
  pendingTOTPSecret?: string;
  awaitingTOTP?: boolean;
  mustResetPassword?: boolean;
  createdAt?: number;
  lastActivity?: number;
  ipAddress?: string | null;
}
```

Note: Use `string` (not the Prisma Role enum) to keep the type declaration file free of Prisma imports. Route handlers will use the Role enum values ('NORMAL' | 'PM' | 'MANAGER' | 'ADMIN') at runtime.

**Acceptance:** TypeScript compilation of the types file succeeds.

### Task 2: Create requireRole middleware factory

**Scope:** `backend/src/middleware/auth.ts`

**Action:**

1. Define the role hierarchy as a constant:
   ```typescript
   const ROLE_HIERARCHY: Record<string, number> = {
     NORMAL: 1,
     PM: 2,
     MANAGER: 3,
     ADMIN: 4,
   };
   ```

2. Create `requireRole` factory function:
   ```typescript
   export function requireRole(minimumRole: string) {
     return (req: Request, res: Response, next: NextFunction) => {
       // Check authentication first
       if (!req.session.userId || !req.session.totpVerified) {
         return res.status(401).json({ error: 'Not authenticated' });
       }

       const userLevel = ROLE_HIERARCHY[req.session.role] ?? 0;
       const requiredLevel = ROLE_HIERARCHY[minimumRole] ?? Infinity;

       if (userLevel < requiredLevel) {
         return res.status(403).json({ error: 'Forbidden' });
       }

       next();
     };
   }
   ```

3. Remove the old `requireAdmin` function. Add a backwards-compatible re-export for any route files that haven't been migrated yet:
   ```typescript
   /** @deprecated Use requireRole('ADMIN') instead */
   export const requireAdmin = requireRole('ADMIN');
   ```
   This ensures the codebase compiles even before Plan 04 migrates route files.

4. `requireAuth` and `requirePendingTOTP` remain unchanged — they don't check role/admin status.

**Acceptance:** `requireRole('PM')` middleware blocks NORMAL users (403), allows PM/MANAGER/ADMIN. `requireRole('ADMIN')` blocks all non-ADMIN users. `requireAuth` still works for basic auth checks.

### Task 3: Add session invalidation function

**Scope:** `backend/src/services/session.ts`

**Action:**

Add a new exported function `invalidateUserSessions` that deletes all Redis sessions for a given userId:

```typescript
/**
 * Invalidate all active sessions for a user (force re-login)
 * Used when admin changes a user's role
 * @param userId - User ID whose sessions should be destroyed
 * @returns Number of sessions destroyed
 */
export async function invalidateUserSessions(userId: string): Promise<number> {
  try {
    const keys = await redisClient.keys('layer8:sess:*');
    let destroyed = 0;

    for (const key of keys) {
      const sessionData = await redisClient.get(key);
      if (!sessionData) continue;

      try {
        const parsed = JSON.parse(sessionData);
        if (parsed.userId === userId) {
          await redisClient.del(key);
          destroyed++;
        }
      } catch {
        continue;
      }
    }

    return destroyed;
  } catch (error) {
    console.error('[session service] Error invalidating user sessions:', error);
    return 0;
  }
}
```

**Acceptance:** Calling `invalidateUserSessions(userId)` deletes all Redis session keys containing that userId. Function is exported and importable from other modules.
