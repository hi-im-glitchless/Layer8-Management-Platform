## Objective

Create the frontend RBAC foundation: a shared role type and hierarchy utility, updated auth types replacing `isAdmin` with `role`, and an enhanced `useAuth()` hook that exposes role-based access checks. Also update admin types to use role throughout.

## Context

- `@frontend/src/features/auth/types.ts` -- User interface has `isAdmin: boolean` at line 10; needs `role: Role`
- `@frontend/src/features/auth/hooks.ts` -- useAuth() returns `isAdmin: query.data?.isAdmin ?? false` at line 28; needs role + hasRole
- `@frontend/src/features/admin/types.ts` -- AdminUser has `isAdmin: boolean` (line 3), CreateUserRequest has `isAdmin?: boolean` (line 25), UpdateUserRequest has `isAdmin?: boolean` (line 28)
- New file `frontend/src/lib/rbac.ts` needed for shared role type and utility

## Tasks

### Task 1: Create shared role types and hasRole utility

**Scope:** `frontend/src/lib/rbac.ts` (new file)

**Action:**

Create a new file with the Role type, hierarchy, and utility:

```typescript
export type Role = 'NORMAL' | 'PM' | 'MANAGER' | 'ADMIN';

export const ROLE_HIERARCHY: Record<Role, number> = {
  NORMAL: 1,
  PM: 2,
  MANAGER: 3,
  ADMIN: 4,
};

export const ROLE_LABELS: Record<Role, string> = {
  NORMAL: 'Normal',
  PM: 'Project Manager',
  MANAGER: 'Manager',
  ADMIN: 'Admin',
};

/**
 * Check if a user role meets the minimum required role
 * Hierarchical: ADMIN > MANAGER > PM > NORMAL
 */
export function hasRole(userRole: Role | undefined, minimumRole: Role): boolean {
  if (!userRole) return false;
  return (ROLE_HIERARCHY[userRole] ?? 0) >= (ROLE_HIERARCHY[minimumRole] ?? Infinity);
}
```

**Acceptance:** File exists, exports are importable, `hasRole('MANAGER', 'PM')` returns true, `hasRole('NORMAL', 'PM')` returns false.

### Task 2: Update auth types

**Scope:** `frontend/src/features/auth/types.ts`

**Action:**

1. Import Role type:
   ```typescript
   import type { Role } from '@/lib/rbac';
   ```

2. Replace `isAdmin: boolean` with `role: Role` in the User interface:
   ```typescript
   export interface User {
     id: string;
     username: string;
     displayName?: string | null;
     avatarUrl?: string | null;
     role: Role;
     totpEnabled: boolean;
     createdAt?: string;
     lastLoginAt?: string | null;
   }
   ```

3. No changes needed to LoginResponse, TOTPSetupResponse, or other response types.

**Acceptance:** User interface has `role: Role`. No `isAdmin` field.

### Task 3: Update useAuth hook

**Scope:** `frontend/src/features/auth/hooks.ts`

**Action:**

1. Import `hasRole` and `Role` from `@/lib/rbac`:
   ```typescript
   import { hasRole as checkRole, type Role } from '@/lib/rbac';
   ```

2. Update the `useAuth()` return value to replace `isAdmin` with role-aware properties:
   ```typescript
   return {
     user: query.data,
     isLoading: query.isLoading,
     isAuthenticated: !!query.data && !query.error,
     role: query.data?.role ?? 'NORMAL',
     hasRole: (minimumRole: Role) => checkRole(query.data?.role, minimumRole),
     refetch: query.refetch,
     error: query.error,
   };
   ```

3. Remove the `isAdmin` property entirely.

**Acceptance:** `useAuth()` returns `role` (string) and `hasRole` (function). No `isAdmin` property. Consumers can call `hasRole('ADMIN')` to check admin status.

**Scope:** `frontend/src/features/admin/types.ts`
