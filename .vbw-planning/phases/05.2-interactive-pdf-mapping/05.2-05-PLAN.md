## Objective
Wire all Phase 5.2 components together in the StepAnalysis page. Replace the tabbed Preview/Mappings layout with a unified PDF-first view: InteractivePdfViewer with selection overlays, StructureBrowser sidebar toggle, batch chat panel, coverage counter, and KB badge. The annotated preview shifts to green-only shading (no yellow gap highlighting). This is the final integration plan that delivers the complete interactive mapping experience. Covers acceptance criteria #3 (chat message), #7 (re-describe rejected), #8 (batch corrections), #9 (coverage counter), #10 (KB badge animates), #11 (KB stores per type).

### Task 2: Chat Panel Batch Integration
**File(s):** `frontend/src/features/adapter/components/StepAnalysis.tsx`
**Action:** Modify
**Details:**
- Wire chat SSE events to selection state:
  1. When `selection_mapping` events arrive (via chat.selectionMappings), call `updateSelectionMapping()` on the corresponding selection
  2. When `batch_complete` arrives, show toast: "N mappings resolved -- review and confirm"
  3. When `mapping_update` arrives (existing event, backward compat), update mappingPlan as before
- Chat placeholder text update: "Describe your selections, e.g. '#1 is the executive summary, #2 is findings'"
- After user confirms selections, add context to chat: display a system-style message showing what was confirmed
- Re-mapping flow:
  1. User sees orange (rejected) selections on PDF
  2. User types: "#3 should be the impact section, #5 is scope"
  3. Chat detects #N references for rejected selections
  4. LLM re-maps only those selections
  5. New selection_mapping events update the orange selections back to resolved-pending (blue) with new mappings
  6. User can accept/reject again
- Ensure chat.clearSelectionMappings() is called when starting a new batch
**Commit:** `feat(adapter-ui): wire batch chat events to selection state for mapping and re-mapping`
