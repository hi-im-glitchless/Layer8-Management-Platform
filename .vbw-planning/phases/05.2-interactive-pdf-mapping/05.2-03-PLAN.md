## Objective
Build the LLM prompt infrastructure and backend changes needed for the batch mapping workflow. A new prompt builder accepts numbered selections and instructs the LLM to map each to a GW field. The chat endpoint is extended to detect batch selection references (#N) and use the specialized prompt. New SSE events deliver per-selection mapping results to the frontend. Covers acceptance criteria #3 (free-form chat), #4 (LLM resolves all), #7 (re-describe rejected), #8 (batch corrections).

## Context
- @sanitization-service/app/services/analysis_prompt.py -- Existing prompt builder. New batch prompt is a separate function, not a modification of Pass 1 prompt.
- @sanitization-service/app/models/adapter.py -- Add new Pydantic models for batch mapping request/response
- @sanitization-service/app/routes/adapter.py -- Add new validation endpoint
- @backend/src/services/templateAdapter.ts -- processChatFeedback() generator. Extend to detect batch format and switch prompt.
- @backend/src/routes/templateAdapter.ts -- Chat SSE route. Add new event types.
- @frontend/src/features/adapter/types.ts -- ChatSSEEvent union. Add new event variants.
- Decision #3: Free-form natural language -- user types "#1 is X, #2 is Y", LLM parses references
- Decision #6: Selection stays numbered, turns orange; user re-describes in chat, LLM re-maps only specified
- Decision #7: Same batch pattern as initial description -- multiple corrections in one message

### Task 2: Python Batch Mapping Validation
**File(s):** `sanitization-service/app/models/adapter.py`, `sanitization-service/app/routes/adapter.py`
**Action:** Modify
**Details:**
- Add to `adapter.py`:
  ```python
  class BatchSelectionInput(BaseModel):
      selection_number: int
      text: str
      paragraph_index: int

  class BatchMappingRequest(BaseModel):
      llm_response: str
      selections: list[BatchSelectionInput]
      template_type: TemplateType
      language: TemplateLanguage

  class BatchMappingEntry(BaseModel):
      selection_number: int
      gw_field: str
      marker_type: str
      confidence: float = Field(ge=0, le=1)
      rationale: str = ""

  class BatchMappingResponse(BaseModel):
      valid: bool
      mappings: list[BatchMappingEntry] = Field(default_factory=list)
      errors: list[str] = Field(default_factory=list)
  ```
- Add `POST /adapter/validate-batch-mapping` endpoint:
  1. Parse LLM response as JSON (strip markdown fences)
  2. Validate each entry: selectionNumber must match an input selection, gwField must be non-empty
  3. Validate gwField against known GW field set (warning if unknown, not error)
  4. Return BatchMappingResponse with validated mappings
**Commit:** `feat(adapter): add batch mapping validation endpoint and Pydantic models`

### Task 3: Backend Chat Extension for Batch Selections
**File(s):** `backend/src/services/templateAdapter.ts`
**Action:** Modify
**Details:**
- Add `detectBatchSelections(message: string): boolean` helper:
  - Returns true if message contains `#\d+` references (regex: `/#\d+/`)
- Add `parseBatchSelectionNumbers(message: string): number[]` helper:
  - Extracts all `#N` numbers from the message
- Modify `processChatFeedback()`:
  1. Check if message contains batch selection references
  2. If batch detected, read selections from wizard state (new field: `interactiveSelections`)
  3. Determine if this is initial mapping or re-mapping:
     - If referenced selections have status 'pending' -> initial batch mapping
     - If referenced selections have status 'rejected' -> re-mapping flow
  4. Build system prompt via Python batch_mapping_prompt (POST to new endpoint, or inline if simple)
  5. After LLM response, validate via Python /adapter/validate-batch-mapping
  6. Yield new SSE events: `selection_mapping` for each resolved selection
  7. Yield `batch_complete` when all selections are processed
- If message does NOT contain batch references, fall through to existing chat behavior (backward compatible)
- Add `interactiveSelections` to WizardState interface in wizardState.ts
**Commit:** `feat(adapter): extend chat feedback to detect batch selections and use batch mapping prompts`
