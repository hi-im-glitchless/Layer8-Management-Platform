## Objective
Build the inline mapping overlay components that appear on the PDF at each selection's location after the LLM resolves them. Each overlay shows the proposed GW field mapping with accept/reject controls. Accepting turns the selection green; rejecting turns it orange and retains its number for re-description. A "Confirm All" button allows batch confirmation. Covers acceptance criteria #5 (inline overlays with accept/reject), #6 (confirmed=green, rejected=orange, retain number), and partially #7 (rejected stay numbered for re-description).

## Context
- @frontend/src/features/adapter/components/InteractivePdfViewer.tsx -- Plan 01 creates this with selection badges. This plan replaces pending badges with full overlay cards after LLM resolution.
- @frontend/src/features/adapter/hooks.ts -- useSelectionState from Plan 01 provides confirm/reject actions. useAdapterChat from Plan 03 provides selectionMappings.
- @frontend/src/features/adapter/types.ts -- SelectionEntry with status, gwField, markerType from Plan 01. BatchMappingResult from Plan 03.
- Decision #4: Inline on PDF with accept/reject per mapping -- overlays at selection locations, batch confirm-all
- Decision #6: Selection stays numbered, turns orange; user re-describes in chat
- Decision #8: Three states: blue (pending), green (confirmed), orange (rejected/needs correction)
