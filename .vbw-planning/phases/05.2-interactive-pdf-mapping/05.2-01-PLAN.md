## Objective
Build the core frontend state management for tracking numbered text selections on the PDF, plus the custom mouseup handler that captures text selections from the pdfjs text layer. This is the foundational building block that Plans 4 and 5 depend on for overlay positioning and chat integration. Covers acceptance criteria #1 (select text passages, auto-numbered).

## Context
- @frontend/src/features/adapter/types.ts -- Add new selection types alongside existing MappingEntry/MappingPlan types
- @frontend/src/components/ui/pdf-preview.tsx -- Existing PdfPreview with react-pdf Document/Page, continuous scroll, text layer enabled. Will be extended with a new InteractivePdfViewer wrapper.
- @frontend/src/features/adapter/components/StepAnalysis.tsx -- Parent component that will consume selection state (wired in Plan 5)
- Decision #1: Select + auto-add -- highlight text, immediately added as next numbered selection with toast
- Decision #8: Three visual states -- blue (pending), green (confirmed), orange (rejected/needs correction)

**Details:**

  interface SelectionEntry {

### Task 2: Selection State Hook
**File(s):** `frontend/src/features/adapter/hooks.ts`
**Action:** Modify
**Details:**
- Add `useSelectionState()` custom hook using useReducer:
  - State: `{ selections: SelectionEntry[]; nextNumber: number; counter: { confirmed: number; total: number } }`
  - `addSelection(entry: Omit<SelectionEntry, 'id' | 'selectionNumber' | 'status' | 'gwField' | 'markerType' | 'confidence'>)`: auto-assigns next number, status='pending'
  - `removeSelection(id: string)`: removes by id, does NOT renumber (numbers are stable)
  - `confirmSelection(id: string)`: sets status='confirmed'
  - `rejectSelection(id: string)`: sets status='rejected'
  - `updateSelectionMapping(selectionNumber: number, gwField: string, markerType: string, confidence: number)`: sets mapping fields
  - `resetSelections()`: clears all
  - `getPendingSelections()`: returns selections with status='pending'
  - `getRejectedSelections()`: returns selections with status='rejected'
  - Counter computed from selections array: confirmed count and total count
- Export hook and types
**Commit:** `feat(adapter): implement useSelectionState hook with auto-numbering and status tracking`

### Task 4: PDF Text Selection Handler
**File(s):** `frontend/src/features/adapter/components/InteractivePdfViewer.tsx`
**Action:** Modify
**Details:**
- Implement `handleMouseUp` event handler:
  1. Get `window.getSelection()` -- if no selection or empty, return
  2. Get selected text via `selection.toString().trim()`
  3. Verify selection is within the pdf viewer container (check parent nodes)
  4. Extract bounding rect via `selection.getRangeAt(0).getBoundingClientRect()`
  5. Convert absolute rect to relative coordinates within the pdf container
  6. Determine page number from the closest `.react-pdf__Page` ancestor's `data-page-number` attribute
  7. Estimate paragraph index by finding the closest `span[role="presentation"]` in the text layer and counting its position among siblings
  8. Call `onTextSelected({ text, pageNumber, boundingRect: relativeRect, paragraphIndex })`
  9. Clear the browser selection after capture via `selection.removeAllRanges()`
- Attach handler to the pdf container div via `onMouseUp`
- Debounce/guard against rapid double-selections (100ms cooldown)
**Commit:** `feat(adapter-ui): implement PDF text selection handler with coordinate extraction`
